# 领域驱动设计概览

领域驱动设计（DDD）是 Eric Evans 提出的一套软件设计方法论，旨在通过深入理解业务领域来驱动软件架构设计。

## 核心理念

- **领域模型**: 软件的核心是对业务领域的准确建模
- **通用语言**: 开发团队与业务专家使用统一的术语沟通
- **限界上下文**: 将复杂系统分解为多个自治的模型边界

## 战略设计

- 限界上下文划分
- 上下文映射（Context Map）
- 核心域 / 支撑域 / 通用域

## 战术设计

- 实体（Entity）与值对象（Value Object）
- 聚合（Aggregate）与聚合根
- 领域事件（Domain Event）
- 仓储（Repository）与工厂（Factory）
