GORM 是 Go 语言中主流的 ORM 框架，通过面向对象的方式操作数据库，无需手写大量 SQL 语句。本文从基础到核心功能，用通俗易懂的方式带你入门，并提供可直接运行的代码示例。

---

## GORM 基础准备

### 安装 GORM
首先需要安装 GORM 核心包和对应的数据库驱动（以 MySQL 为例）：

```bash
# 安装 GORM v2 核心包
go get gorm.io/gorm
# 安装 MySQL 驱动
go get gorm.io/driver/mysql
```

### 核心概念

| 概念 | 说明 |
|------|------|
| **Model** | Go 结构体，映射数据库表（字段对应列） |
| **DB 实例** | GORM 的核心操作对象，用于执行增删改查 |
| **CRUD** | GORM 封装了完整的增删改查方法，无需手写 SQL |

## 连接数据库（以 MySQL 为例）

```go
package main

import (
  "gorm.io/driver/mysql"
  "gorm.io/gorm"
  "gorm.io/gorm/logger"
  "log"
  "time"
)

// 定义 Model（映射数据库表）
// GORM 默认规则：结构体名蛇形复数为表名（User -> users），ID 为主键
type User struct {
  gorm.Model        // 内置字段：ID, CreatedAt, UpdatedAt, DeletedAt（软删除）
  Name      string  `gorm:"column:name;type:varchar(50);not null"` // 自定义列属性
  Age       int     `gorm:"column:age;default:0"`
  Email     string  `gorm:"column:email;uniqueIndex"` // 唯一索引
}

func main() {
  // 1. 配置 MySQL 连接信息
  // dsn 格式：用户名:密码@tcp(地址:端口)/数据库名?charset=utf8mb4&parseTime=True&loc=Local
  dsn := "root:123456@tcp(127.0.0.1:3306)/test_db?charset=utf8mb4&parseTime=True&loc=Local"

  // 2. 连接数据库，创建 DB 实例
  db, err := gorm.Open(mysql.Open(dsn), &gorm.Config{
    Logger: logger.Default.LogMode(logger.Info), // 打印 SQL 日志（方便调试）
  })
  if err != nil {
    log.Fatalf("数据库连接失败：%v", err)
  }

  // 3. 自动迁移（根据 Model 创建/更新表结构，仅新增字段，不会删除）
  err = db.AutoMigrate(&User{})
  if err != nil {
    log.Fatalf("表迁移失败：%v", err)
  }

  // 后续 CRUD 操作都基于这个 db 实例...
}
```

**关键解释**：
- `gorm.Model` 是 GORM 内置的结构体，包含主键、创建时间、更新时间、软删除字段，推荐嵌入使用
- `AutoMigrate` 是开发阶段常用功能，生产环境建议手动建表
- DSN 中的 `parseTime=True` 必须加，否则 GORM 无法处理时间类型

## 核心 CRUD 操作

### 新增（Create）

```go
func createUser(db *gorm.DB) {
  // 单条新增
  user := User{Name: "张三", Age: 25, Email: "zhangsan@test.com"}
  result := db.Create(&user) // 传指针，新增后 user.ID 会被赋值
  if result.Error != nil {
    log.Printf("新增失败：%v", result.Error)
    return
  }
  log.Printf("新增成功，ID：%d", user.ID)

  // 批量新增
  users := []User{{Name: "李四", Age: 26}, {Name: "王五", Age: 27}}
  db.Create(&users)
}
```

### 查询（Read）

```go
func queryUser(db *gorm.DB) {
  // 1. 根据主键查询单条
  var user User
  db.First(&user, 1) // 查询 ID=1 的用户，First 会返回第一条
  log.Printf("主键查询：%+v", user)

  // 2. 条件查询
  var user2 User
  db.Where("name = ?", "李四").First(&user2) // WHERE name = '李四'
  log.Printf("条件查询：%+v", user2)

  // 3. 批量查询
  var users []User
  db.Where("age > ?", 25).Find(&users) // 查询年龄>25 的所有用户
  log.Printf("批量查询：%+v", users)

  // 4. 模糊查询
  db.Where("name LIKE ?", "%张%").Find(&users)
}
```

### 更新（Update）

```go
func updateUser(db *gorm.DB) {
  // 1. 更新单条（推荐：先查询再更新，避免全字段更新）
  var user User
  db.First(&user, 1)
  user.Age = 28
  db.Save(&user) // Save 会更新所有字段

  // 2. 条件更新（指定字段）
  db.Model(&User{}).Where("name = ?", "李四").Update("age", 29)

  // 3. 批量更新多个字段
  db.Model(&User{}).Where("age > ?", 25).Updates(map[string]interface{}{
    "age":  30,
    "name": "更新后的名字",
  })
}
```

### 删除（Delete）

```go
func deleteUser(db *gorm.DB) {
  // 1. 软删除（默认，仅修改 DeletedAt 字段，数据仍在）
  db.Delete(&User{}, 1) // 删除 ID=1 的用户

  // 2. 物理删除（真正删除数据）
  db.Unscoped().Delete(&User{}, 2) // 物理删除 ID=2 的用户

  // 3. 条件删除
  db.Where("age < ?", 25).Delete(&User{})
}
```

## 关联查询（一对一/一对多）

以「用户-订单」一对多为例：

```go
// 定义订单 Model
type Order struct {
  gorm.Model
  UserID uint   // 外键，关联 User 的 ID
  Title  string // 订单标题
  Price  float64
}

// 1. 先迁移表
db.AutoMigrate(&Order{})

// 2. 查询用户并关联订单（预加载）

log.Printf("用户及订单：%+v", user)

// 3. 给用户创建订单
db.Create(&Order{UserID: 1, Title: "手机订单", Price: 2999})
```

## 事务处理

事务是保证数据一致性的关键，GORM 提供了两种事务使用方式，覆盖不同场景。

### 自动事务（推荐，简化错误处理）

GORM 会自动管理事务的开启、提交和回滚，只需传入一个包含业务逻辑的函数：

```go
// 自动事务示例：新增用户并创建订单，要么都成功，要么都失败
func autoTransaction(db *gorm.DB) error {
  // 开启自动事务
  err := db.Transaction(func(tx *gorm.DB) error {
    // 1. 新增用户
    user := User{Name: "赵六", Age: 30, Email: "zhaoliu@test.com"}
    if err := tx.Create(&user).Error; err != nil {
      // 返回错误会触发事务回滚
      return err
    }

    // 2. 新增订单（关联刚创建的用户）
    order := Order{UserID: user.ID, Title: "电脑订单", Price: 5999}
    if err := tx.Create(&order).Error; err != nil {
      return err
    }

    // 返回 nil 则提交事务
    return nil
  })

  if err != nil {
    log.Printf("事务执行失败：%v", err)
    return err
  }
  log.Println("事务执行成功")
  return nil
}
```

### 手动事务（灵活控制）

适合需要分步操作、手动控制提交/回滚的场景：

```go
func manualTransaction(db *gorm.DB) {
  // 1. 开启事务
  tx := db.Begin()
  defer func() {
    // 捕获 panic，避免事务未处理
    if r := recover(); r != nil {
      tx.Rollback()
    }
  }()

  // 2. 检查开启是否失败
  if tx.Error != nil {
    log.Printf("开启事务失败：%v", tx.Error)
    return
  }

  // 3. 执行业务操作（所有操作使用 tx 而非 db）
  if err := tx.Create(&User{Name: "孙七", Age: 31}).Error; err != nil {
    tx.Rollback() // 失败回滚
    log.Printf("新增失败：%v", err)
    return
  }

  // 4. 提交事务
  if err := tx.Commit().Error; err != nil {
    tx.Rollback()
    log.Printf("提交事务失败：%v", err)
    return
  }
  log.Println("手动事务提交成功")
}
```

## 钩子函数（生命周期回调）

钩子函数是 GORM 在 CRUD 操作的特定阶段自动执行的方法，用于统一处理逻辑（如数据校验、字段赋值），无需在业务代码中重复编写。

### 常用钩子示例（以 User 模型为例）

```go
// 重写 User 模型，添加钩子函数
type User struct {
  gorm.Model
  Name      string `gorm:"column:name;type:varchar(50);not null"`
  Age       int    `gorm:"column:age;default:0"`
  Email     string `gorm:"column:email;uniqueIndex"`
  CreateBy  string `gorm:"column:create_by"` // 新增时自动赋值
}

// 1. 新增前钩子：BeforeCreate（执行 Create 前触发）
func (u *User) BeforeCreate(tx *gorm.DB) error {
  // 示例：自动填充创建人
  u.CreateBy = "system"
  // 数据校验：年龄不能小于 0
  if u.Age < 0 {
    return errors.New("年龄不能为负数")
  }
  return nil
}

// 2. 更新前钩子：BeforeUpdate（执行 Update/Save 前触发）
func (u *User) BeforeUpdate(tx *gorm.DB) error {
  // 示例：禁止修改创建人
  tx.Statement.Omit("create_by") // 忽略 create_by 字段的更新
  return nil
}

// 3. 删除前钩子：BeforeDelete
func (u *User) BeforeDelete(tx *gorm.DB) error {
  // 示例：禁止删除年龄大于 60 的用户
  if u.Age > 60 {
    return errors.New("禁止删除老年用户")
  }
  return nil
}
```

**核心说明**：
- 钩子函数必须是模型的方法，且接收 `*gorm.DB` 参数，返回 `error`
- 返回非 nil 错误会中断当前操作（如新增前钩子返回错误，数据不会插入）
- 常用钩子：`BeforeCreate`/`AfterCreate`、`BeforeUpdate`/`AfterUpdate`、`BeforeDelete`/`AfterDelete`

## 高级查询（复杂条件/关联/分页）

### 复杂条件查询（多条件、排序、分组）

```go
func complexQuery(db *gorm.DB) {
  var users []User

  // 示例1：多条件组合（AND/OR）
  db.Where("age > ? AND name LIKE ?", 25, "%张%").
    Or("email LIKE ?", "%test%"). // OR 条件
    Find(&users)

  // 示例2：排序 + 分页
  // Offset：跳过前 N 条，Limit：取 N 条（分页公式：Offset=(页码-1)*页大小）
  db.Order("age DESC, created_at ASC"). // 按年龄降序、创建时间升序
    Offset(0).Limit(10). // 第 1 页，每页 10 条
    Find(&users)

  // 示例3：分组 + 统计
  type Result struct {
    Age   int
    Count int
  }
  var results []Result
  db.Model(&User{}).
    Select("age, count(*) as count").
    Group("age"). // 按年龄分组
    Having("count(*) > ?", 1). // 筛选分组结果（每组数量>1）
    Scan(&results) // 扫描到自定义结构体
  log.Printf("分组统计结果：%+v", results)
}
```

### 关联查询进阶（预加载筛选、嵌套预加载）

```go
func advancedPreload(db *gorm.DB) {
  var user User

  // 示例1：预加载时筛选关联数据（只加载用户的高价订单）
  db.Preload("Orders", "price > ?", 3000). // 只加载价格>3000 的订单
    First(&user, 1)

  // 示例2：嵌套预加载（如果订单还有子表，如 OrderItem）
  // 假设 Order 有关联的 OrderItem 模型，预加载用户→订单→订单项
  db.Preload("Orders.OrderItems").First(&user, 1)

  // 示例3：自定义预加载 SQL
  db.Preload("Orders", func(tx *gorm.DB) *gorm.DB {
    return tx.Order("price DESC").Limit(2) // 只取 2 个最贵的订单
  }).First(&user, 1)
}
```

## 原生 SQL 执行

GORM 支持执行原生 SQL，应对 ORM 无法覆盖的复杂场景（如多表联查、复杂函数）：

```go
func rawSQL(db *gorm.DB) {
  // 1. 原生查询（扫描到结构体）
  var users []User
  db.Raw("SELECT * FROM users WHERE age > ?", 25).Scan(&users)

  // 2. 原生更新/删除（执行后获取影响行数）
  result := db.Exec("UPDATE users SET age = ? WHERE name = ?", 30, "张三")
  log.Printf("影响行数：%d", result.RowsAffected)

  // 3. SQL 参数绑定（避免 SQL 注入）
  db.Raw("SELECT * FROM users WHERE name IN ?", []string{"张三", "李四"}).Scan(&users)
}
```

## 批量操作优化

批量操作是提升性能的关键，GORM 提供了针对性的优化方案：

```go
func batchOperation(db *gorm.DB) {
  // 1. 批量新增（单次 SQL，性能最优）
  users := []User{
    {Name: "批量1", Age: 20},
    {Name: "批量2", Age: 21},
  }
  db.CreateInBatches(users, 100) // 每 100 条批量插入一次

  // 2. 批量更新（CASE WHEN 语法，单次 SQL）
  var updateUsers = []User{
    {Model: gorm.Model{ID: 1}, Age: 30},
    {Model: gorm.Model{ID: 2}, Age: 31},
  }
  db.Save(&updateUsers) // GORM 自动转为批量更新

  // 3. 批量删除
  db.Delete(&User{}, []int{1, 2, 3}) // 删除 ID 为 1、2、3 的用户（单次 SQL）
}
```

## 读写分离（生产环境必备）

GORM 支持读写分离，提升数据库并发能力，需配合 `gorm.io/plugin/dbresolver` 插件：

```go
// 1. 安装插件
// go get gorm.io/plugin/dbresolver

// 2. 配置读写分离
func readWriteSplit() {
  // 主库（写）
  dsnMaster := "root:123456@tcp(127.0.0.1:3306)/test_db?charset=utf8mb4&parseTime=True&loc=Local"
  // 从库（读）
  dsnSlave := "root:123456@tcp(127.0.0.1:3307)/test_db?charset=utf8mb4&parseTime=True&loc=Local"

  db, err := gorm.Open(mysql.Open(dsnMaster), &gorm.Config{})
  if err != nil {
    log.Fatal(err)
  }

  // 配置读写分离
  _ = db.Use(dbresolver.Register(dbresolver.Config{
    // 读操作使用从库
    Replicas: []gorm.Dialector{mysql.Open(dsnSlave)},
    // 策略：随机选择从库
    Policy: dbresolver.RandomPolicy{},
  }).SetMaxOpenConns(100).SetMaxIdleConns(20))

  // 后续操作：写操作自动走主库，读操作自动走从库
  db.Create(&User{Name: "主库写入"}) // 写→主库
  var user User
  db.First(&user, 1) // 读→从库
}
```

## 乐观锁（解决并发更新冲突）

乐观锁是生产环境中解决「并发更新同一数据」的核心方案，GORM 通过插件原生支持，无需手写版本控制逻辑。

### 使用步骤

```go
// 第一步：安装乐观锁插件
// go get gorm.io/plugin/optimisticlock

import (
  "gorm.io/plugin/optimisticlock"
)

// 第二步：模型中添加 Version 字段（必须是 int 类型，标签指定 optimistic_lock）
type Product struct {
  gorm.Model
  Name    string `gorm:"varchar(50)"`
  Stock   int    `gorm:"default:0"`
  Version int    `gorm:"optimistic_lock"` // 乐观锁版本字段
}

// 第三步：注册插件并使用
func optimisticLockDemo(db *gorm.DB) {
  // 注册乐观锁插件
  _ = db.Use(optimisticlock.New())

  // 迁移表
  _ = db.AutoMigrate(&Product{})

  // 模拟并发更新库存
  // 场景：两个请求同时扣减同一个商品的库存，乐观锁保证只有一个成功
  var product Product
  // 请求1查询商品
  db.First(&product, 1)
  product.Stock -= 1
  // 请求2（模拟并发）也查询商品
  var product2 Product
  db.First(&product2, 1)
  product2.Stock -= 1

  // 请求1更新（版本匹配，成功，Version 自动+1）
  res1 := db.Save(&product)
  log.Printf("请求1更新结果：%v，影响行数：%d", res1.Error, res1.RowsAffected) // 成功，RowsAffected=1

  // 请求2更新（版本已被请求1修改，失败，RowsAffected=0）
  res2 := db.Save(&product2)
  log.Printf("请求2更新结果：%v，影响行数：%d", res2.Error, res2.RowsAffected) // 无错误，但 RowsAffected=0
}
```

**核心说明**：
- 乐观锁通过「版本号」控制，每次更新会自动检查版本：`UPDATE products SET ..., version=version+1 WHERE id=? AND version=?`
- 若版本不匹配，更新无影响（`RowsAffected=0`），业务中可提示「数据已更新，请刷新重试」
- 适合读多写少的场景（如商品库存、用户积分），性能优于悲观锁（无需加锁）

## 命名参数（提升复杂 SQL 可读性）

GORM 支持命名参数替代 `?` 占位符，尤其在复杂 SQL 中，能大幅提升代码可读性，避免参数顺序错误。

```go
func namedParamsDemo(db *gorm.DB) {
  var users []User

  // 传统占位符：参数顺序易出错
  db.Where("age > ? AND name LIKE ?", 25, "%张%").Find(&users)

  // 命名参数：通过:key 指定，参数用 map 传递，顺序无关
  db.Where("age > :age AND name LIKE :name", map[string]interface{}{
    "age":  25,
    "name": "%张%",
  }).Find(&users)

  // 原生 SQL 中也支持
  db.Raw(`
    SELECT * FROM users
    WHERE age > :age
    AND created_at > :create_time
  `, map[string]interface{}{
    "age":         25,
    "create_time": "2024-01-01",
  }).Scan(&users)
}
```

## 字段权限控制（精细管控字段操作）

通过模型标签或 `Omit/Select`，可以精细化控制字段的「可创建、可更新、可查询」权限，避免敏感字段被篡改。

### 模型标签方式（全局控制）

```go
type Admin struct {
  gorm.Model
  Username string `gorm:"not null;unique"`
  Password string `gorm:"not null;<-:create"` // 仅创建时可赋值，更新时忽略
  Role     string `gorm:"not null;->:false"`  // 禁止查询（查询结果中不返回该字段）
  Status   int    `gorm:"default:1;<-:false"` // 禁止更新（只能通过代码手动修改）
}

// 测试：
func fieldPermissionDemo(db *gorm.DB) {
  // 1. 创建：Password 可赋值，正常插入
  admin := Admin{Username: "admin", Password: "123456", Role: "super", Status: 1}
  db.Create(&admin) // 成功

  // 2. 更新：尝试修改 Password 和 Status（会被忽略）
  admin.Password = "654321"
  admin.Status = 0
  db.Save(&admin) // Password 和 Status 不会被更新

  // 3. 查询：Role 字段不会返回（值为默认空）
  var queryAdmin Admin
  db.First(&queryAdmin, admin.ID)
  log.Printf("查询结果：%+v", queryAdmin) // Role=""
}
```

### 动态控制（临时指定字段）

```go
// 只更新指定字段（推荐：避免全字段更新）
db.Model(&User{}).Select("age").Where("id=?", 1).Updates(map[string]interface{}{"age": 30, "name": "无效"})

// 忽略指定字段（临时跳过）
db.Model(&User{}).Omit("email", "create_by").Save(&user)
```

## 自定义数据类型（适配特殊字段）

GORM 支持自定义数据类型（如 JSON、枚举、IP 地址等），解决原生类型无法满足的场景。

### 示例：JSON 类型字段

```go
import (
  "database/sql/driver"
  "encoding/json"
  "errors"
)

// 1. 定义自定义 JSON 类型
type UserInfo struct {
  Hobby  []string `json:"hobby"`
  Address string  `json:"address"`
}

// 2. 实现 driver.Valuer 和 sql.Scanner 接口（GORM 要求）
func (u UserInfo) Value() (driver.Value, error) {
  // 序列化：转为 JSON 字符串存入数据库
  return json.Marshal(u)
}

func (u *UserInfo) Scan(v interface{}) error {
  // 反序列化：从数据库取出 JSON 字符串转为结构体
  bytes, ok := v.([]byte)
  if !ok {
    return errors.New("类型断言失败")
  }
  return json.Unmarshal(bytes, u)
}

// 3. 模型中使用
type CustomUser struct {
  gorm.Model
  Name  string   `gorm:"varchar(50)"`
  Info  UserInfo `gorm:"type:json"` // 字段类型为 json
}

// 4. 使用示例
func customTypeDemo(db *gorm.DB) {
  _ = db.AutoMigrate(&CustomUser{})

  // 创建用户（自动序列化为 JSON）
  user := CustomUser{
    Name: "张三",
    Info: UserInfo{
      Hobby:  []string{"篮球", "游戏"},
      Address: "北京市",
    },
  }
  db.Create(&user)

  // 查询用户（自动反序列化）
  var queryUser CustomUser
  db.First(&queryUser, user.ID)
```

## 关键注意事项

| 注意事项 | 说明 |
|---------|------|
| **软删除** | GORM 默认开启软删除，删除操作仅修改 `DeletedAt` 字段，查询时会自动过滤已软删除的数据；如需查询软删除数据，使用 `db.Unscoped().Find(&user)` |
| **字段标签** | 通过 `gorm:"xxx"` 可以自定义列名、类型、索引、默认值等，比如 `gorm:"primaryKey"` 指定主键 |
| **错误处理** | 所有 GORM 操作的返回值都要检查 `Error`，避免忽略异常 |
| **性能优化** | 查询时指定字段：`db.Select("name", "age").Find(&users)`；分页查询：`db.Limit(10).Offset(0).Find(&users)`；避免 N+1 查询：用 `Preload` 预加载关联数据 |
