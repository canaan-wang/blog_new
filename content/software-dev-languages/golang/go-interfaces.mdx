# Go 接口设计原则

Go 语言的接口是隐式实现的，这是与大多数面向对象语言的本质区别。

## 隐式接口

在 Go 中，一个类型只要实现了接口中的所有方法，就自动实现了该接口：

```go
type Writer interface {
    Write(p []byte) (n int, err error)
}

type MyWriter struct{}

func (w *MyWriter) Write(p []byte) (n int, err error) {
    // 实现写入逻辑
    return len(p), nil
}
```

## 接口设计的最佳实践

### 1. 接口应该小而专注

Go 标准库中的接口往往只有一到两个方法：

- `io.Reader` — 只有 `Read` 方法
- `io.Writer` — 只有 `Write` 方法
- `fmt.Stringer` — 只有 `String` 方法

### 2. 在消费者端定义接口

```go
// 在使用接口的包中定义，而非实现者的包中
type UserStore interface {
    GetUser(id string) (*User, error)
}

func NewService(store UserStore) *Service {
    return &Service{store: store}
}
```

### 3. 接受接口，返回具体类型

```go
// Good
func NewHandler(logger Logger) *Handler { ... }

// Avoid
func NewHandler(logger Logger) Handler { ... }
```

## 总结

Go 的接口设计鼓励我们编写松耦合、可测试的代码。遵循小接口、消费者端定义等原则，可以写出更优雅的 Go 代码。
