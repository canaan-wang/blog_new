# Kafka 消息队列核心概念

Apache Kafka 是一个分布式流处理平台，广泛应用于大规模数据管道和实时数据处理场景。

## 核心架构

Kafka 的架构由以下核心组件组成：

- **Broker**: Kafka 服务器节点
- **Topic**: 消息的逻辑分类
- **Partition**: Topic 的物理分片
- **Producer**: 消息生产者
- **Consumer**: 消息消费者
- **Consumer Group**: 消费者组

## 消息模型

```
Producer → Topic (Partition 0, 1, 2...) → Consumer Group
```

每个 Partition 内的消息是有序的，Kafka 通过 offset 来追踪消费进度：

```java
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("group.id", "my-group");

KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
consumer.subscribe(Arrays.asList("my-topic"));

while (true) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
    for (ConsumerRecord<String, String> record : records) {
        System.out.printf("offset=%d, key=%s, value=%s%n",
            record.offset(), record.key(), record.value());
    }
}
```

## 可靠性保证

Kafka 提供三种消息投递语义：

1. **At most once** — 消息可能丢失，但不会重复
2. **At least once** — 消息不会丢失，但可能重复
3. **Exactly once** — 消息不丢失也不重复（需要事务支持）

## 总结

Kafka 通过分区、副本和消费者组等机制，实现了高吞吐、高可用的消息系统。理解这些核心概念是使用 Kafka 的基础。
