Elasticsearch 查询语法基于 JSON 格式，主要分为两大类型：**结构化查询（Query DSL）** 和 **过滤查询（Filter DSL）**。在新版 ES 中已整合到 Query DSL 里，核心区别是 Filter 不计算评分、可缓存，Query 计算评分。

## 核心查询结构

所有 ES 查询都遵循固定的外层结构，最基础的格式如下：

```json
{
  "query": {
    "查询类型": {
      "查询条件": "值"
    }
  },
  "from": 0,
  "size": 10,
  "_source": ["字段1", "字段2"],
  "sort": [
    {"字段名": {"order": "asc"}}
  ]
}
```

| 参数 | 说明 |
|------|------|
| `query` | 核心查询条件 |
| `from` | 分页起始位置，默认 0 |
| `size` | 返回结果数量，默认 10 |
| `_source` | 指定返回的字段，不写返回全部 |
| `sort` | 排序，`asc` 升序，`desc` 降序 |

---

### 范围查询（Range Query）

用于数值、日期等字段的范围匹配：

```json
{
  "query": {
    "range": {
      "price": {
        "gte": 100,
        "lte": 500,
        "boost": 2.0
      },
      "create_time": {
        "gt": "2024-01-01T00:00:00",
        "format": "yyyy-MM-dd'T'HH:mm:ss"
      }
    }
  }
}
```

| 运算符 | 说明 |
|--------|------|
| `gt` | 大于（greater than） |
| `gte` | 大于等于（greater than or equal） |
| `lt` | 小于（less than） |
| `lte` | 小于等于（less than or equal） |
| `boost` | 权重提升，增加该条件的评分占比 |

---

### 通配符查询（Wildcard Query）

支持 `*`（匹配任意字符）和 `?`（匹配单个字符），适合模糊匹配（性能较低，尽量少用）：

```json
{
  "query": {
    "wildcard": {
      "username.keyword": {
        "value": "user*123",
        "case_insensitive": true
      }
    }
  }
}
```

---

### 聚合查询（Aggregation）

不是检索文档，而是对查询结果做统计分析（如分组、求和、平均值），核心是 `aggs` 字段：

```json
{
  "query": {
    "match": {"title": "ES"}
  },
  "aggs": {
    "group_by_status": {
      "terms": {
        "field": "status.keyword",
        "size": 10
      },
      "aggs": {
        "avg_price": {
          "avg": {"field": "price"}
        }
      }
    },
    "max_price": {
      "max": {"field": "price"}
    }
  }
}
```

## 高级查询类型

### 精准短语匹配（Match Phrase）

和 `match` 不同，`match_phrase` 要求查询词**不分词且按顺序完整匹配**，适合精准检索短语（比如固定搭配、专有名词）。

```json
{
  "query": {
    "match_phrase": {
      "content": {
        "query": "Elasticsearch 实战教程",
        "slop": 1
      }
    }
  }
}
```

| 参数 | 说明 |
|------|------|
| `query` | 必须完整匹配的短语（顺序不变） |
| `slop` | 允许词之间的间隔（slop=1 表示词之间可隔 1 个其他词） |

---

### 多字段匹配（Multi Match）

一次性对**多个字段**执行相同的全文检索，避免写多个 `match` 重复逻辑。

```json
{
  "query": {
    "multi_match": {
      "query": "ES 性能优化",
      "fields": ["title^3", "content", "tags"],
      "type": "best_fields"
    }
  }
}
```

| 参数 | 说明 |
|------|------|
| `query` | 查询关键词 |
| `fields` | 要匹配的字段，`^3` 表示 title 字段权重 × 3 |
| `type` | 匹配策略 |

匹配策略说明：

| 策略 | 说明 |
|------|------|
| `best_fields` | 取匹配度最高的字段评分（默认） |
| `most_fields` | 汇总所有匹配字段的评分 |
| `cross_fields` | 跨字段匹配（比如"ES 性能"可能 ES 在 title、性能在 content） |

---

### 地理空间查询（Geo Queries）

针对 `geo_point`（地理坐标）类型字段的查询，适合 LBS（位置服务）场景（比如"查找附近 1 公里的商家"）。

#### 距离范围查询（Geo Distance）

```json
{
  "query": {
    "geo_distance": {
      "distance": "1km",
      "location": {
        "lat": 30.1234,
        "lon": 120.5678
      }
    }
  }
}
```

| 参数 | 说明 |
|------|------|
| `distance` | 距离范围（支持 km/m/mi 等单位） |
| `location` | 地理坐标字段，包含 `lat`（纬度）和 `lon`（经度） |

#### 多边形范围查询（Geo Polygon）

匹配坐标在指定多边形内的文档：

```json
{
  "query": {
    "geo_polygon": {
      "location": {
        "points": [
          {"lat": 30.0, "lon": 120.0},
          {"lat": 30.1, "lon": 120.1},
          {"lat": 30.0, "lon": 120.2}
        ]
      }
    }
  }
}
```

---

### 嵌套文档查询（Nested Query）

ES 中如果字段是 `nested` 类型（存储嵌套 JSON 数组），普通查询无法精准匹配嵌套文档的多字段关联条件，必须用 `nested` 查询。

文档结构示例：

```json
{
  "product": "手机",
  "comments": [
    {"user": "张三", "score": 5, "content": "续航好"},
    {"user": "李四", "score": 2, "content": "发热严重"}
  ]
}
```

查询"评论中 score ≥ 4 且 content 包含'续航'"的商品：

```json
{
  "query": {
    "nested": {
      "path": "comments",
      "query": {
        "bool": {
          "must": [
            {"range": {"comments.score": {"gte": 4}}},
            {"match": {"comments.content": "续航"}}
          ]
        }
      },
      "inner_hits": {}
    }
  }
}
```

| 参数 | 说明 |
|------|------|
| `path` | 指定 nested 字段的路径 |
| `query` | 嵌套文档的查询条件 |
| `inner_hits` | 可选：返回匹配的嵌套文档（而非整个父文档） |

---

### 高亮显示（Highlight）

检索结果中高亮显示匹配的关键词，提升用户体验：

```json
{
  "query": {
    "match": {"content": "ES 优化"}
  },
  "highlight": {
    "fields": {
      "content": {
        "pre_tags": "<em style='color:red'>",
        "post_tags": "</em>",
        "fragment_size": 100
      }
    }
  }
}
```

返回结果示例：

```json
"highlight": {
  "content": ["<em style='color:red'>ES</em> 性能<em style='color:red'>优化</em>的核心是分片配置"]
}
```

| 参数 | 说明 |
|------|------|
| `pre_tags` | 高亮前缀（自定义 HTML 标签） |
| `post_tags` | 高亮后缀 |
| `fragment_size` | 高亮片段长度（只显示匹配词附近 100 个字符） |

---

### 模糊查询（Fuzzy Query）

允许查询词有少量拼写错误（比如"Elasticsearc"匹配"Elasticsearch"），适合容错检索：

```json
{
  "query": {
    "fuzzy": {
      "title": {
        "value": "Elasticsearc",
        "fuzziness": "AUTO",
        "prefix_length": 3
      }
    }
  }
}
```

| 参数 | 说明 |
|------|------|
| `value` | 可能拼写错误的查询词 |
| `fuzziness` | 容错度：`AUTO`（自动适配词长度）或数字（允许编辑的字符数） |
| `prefix_length` | 前 N 个字符必须精准匹配（减少无效匹配） |

---

### 正则表达式查询（Regexp Query）

支持正则表达式匹配，比通配符更灵活（性能更低，仅适合简单场景）：

```json
{
  "query": {
    "regexp": {
      "username.keyword": {
        "value": "user[0-9]{3}",
        "case_insensitive": true
      }
    }
  }
}
```

---

### 空值/非空值查询（Exists Query）

匹配指定字段"存在值"或"不存在值"的文档：

```json
{
  "query": {
    "bool": {
      "must": [{"exists": {"field": "email"}}],
      "must_not": [{"exists": {"field": "delete_time"}}]
    }
  }
}
```

---

### 脚本查询（Script Query）

支持自定义脚本（Painless 语言）实现复杂逻辑查询，适合无法用常规语法实现的场景：

```json
{
  "query": {
    "script": {
      "script": {
        "source": "doc['price'].value * doc['discount'].value < 100",
        "lang": "painless"
      }
    }
  }
}
```

**注意**：脚本查询性能较低，尽量少用；如需频繁使用，建议提前将计算结果存储为新字段。

## 核心概念

### 评分（_score）

Query 类型的查询会计算每个文档的匹配度评分（越高越相关），Filter 类型不计算评分（`_score` 为 0）。

### 字段类型影响

| 类型 | 特点 | 适用查询 |
|------|------|----------|
| `text` | 会分词 | 适合全文检索，用 `match` |
| `keyword` | 不分词 | 适合精确匹配，用 `term` |

### 分页限制

`from + size` 最大默认是 10000，如需查询更多数据，建议用 `scroll` 或 `search_after`。

---

## 性能优化建议

性能优先级：常规查询（match/term/bool） > 模糊/通配符 > 正则/脚本，生产环境尽量优先使用高性能语法。

| 查询类型 | 性能 | 建议 |
|----------|------|------|
| `match`/`term`/`bool` | 高 | 优先使用 |
| `fuzzy`/`wildcard` | 中 | 尽量少用 |
| `regexp`/`script` | 低 | 仅适合简单场景 |
