Redis 概览

一、什么是 Redis

Redis（Remote Dictionary Server）是开源的内存数据结构存储系统，可用作数据库、缓存、消息队列。

核心特点

  • 内存存储：读写性能极高（10万+ QPS）
  • 丰富的数据类型：String、Hash、List、Set、Sorted Set 等
  • 持久化：支持 RDB 快照和 AOF 日志
  • 高可用：主从复制、哨兵、Cluster 集群
  • 单线程：避免竞态条件，简化设计（6.0+ 多线程处理网络IO）

四、持久化

RDB（Redis Database）

  • 定时快照，保存内存数据到二进制文件
  • 优点：文件紧凑，恢复速度快，适合备份
  • 缺点：可能丢失最后一次快照后的数据

AOF（Append Only File）

  • 记录所有写操作命令，重启时重放
  • 优点：数据更安全，可配置每秒/每次写入
  • 缺点：文件较大，恢复速度较慢

混合持久化（Redis 4.0+）

RDB 全量 + AOF 增量，兼顾速度和安全性。

  • 数据冗余，读写分离
  • 手动故障转移

哨兵（Sentinel）

  • 监控主从节点状态
  • 自动故障转移
  • 提供发现服务

Cluster 集群

  • 数据分片（16384 个槽位）
  • 自动故障转移
  • 水平扩展

六、常见问题

缓存穿透

查询不存在的数据，绕过缓存直达数据库。

解决：布隆过滤器、缓存空值。

缓存击穿

热点 key 过期，瞬间大量请求打到数据库。

解决：互斥锁、逻辑过期、永不过期。

缓存雪崩

大量 key 同时过期，数据库压力剧增。

解决：过期时间加随机值、多级缓存、熔断降级。

大 Key 问题

单个 key value 过大，影响性能。

解决：拆分、压缩、本地缓存。

热 Key 问题

单个 key 访问量过高，节点负载不均。

解决：本地缓存、拆分 key、读写分离。

七、最佳实践

  • 合理设置过期时间，避免同时失效
  • Key 设计规范：业务:模块:id（如 user:profile:1001）
  • 控制 Value 大小，避免大 Key
  • 使用连接池，避免频繁创建连接
  • pipeline 批量操作，减少网络往返
  • 慎用 O(N) 命令（KEYS、FLUSHALL 等）
