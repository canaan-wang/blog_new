Redis 发布订阅（Pub/Sub）

一、简介

Redis Pub/Sub 是一种消息通信模式：发布者发送消息到频道，订阅者接收感兴趣频道的消息。实时性强，但消息不持久化。

二、基本命令

订阅

SUBSCRIBE channel1              # 订阅单个频道
SUBSCRIBE news:sports news:tech # 订阅多个频道
PSUBSCRIBE news.*               # 模式订阅，匹配 news. 开头的频道
UNSUBSCRIBE channel1            # 取消订阅
PUNSUBSCRIBE news.*             # 取消模式订阅

发布

PUBLISH channel1 "Hello World"  # 向频道发送消息

查看频道

PUBSUB CHANNELS                 # 列出所有活跃频道
PUBSUB CHANNELS news.*          # 列出匹配模式的活跃频道
PUBSUB NUMSUB channel1          # 查看频道订阅者数量
PUBSUB NUMPAT                   # 查看模式订阅数量

三、Java 实现

// Jedis 订阅者（需独立线程）
public class Subscriber extends JedisPubSub {
    @Override
    public void onMessage(String channel, String message) {
        System.out.println("收到消息 " + channel + ": " + message);
    }
    
    @Override
    public void onSubscribe(String channel, int subscribedChannels) {
        System.out.println("订阅频道: " + channel);
    }
}

// 启动订阅
new Thread(() -> {
    Jedis jedis = new Jedis("localhost");
    jedis.subscribe(new Subscriber(), "news:sports");
}).start();

// 发布消息
Jedis jedis = new Jedis("localhost");
jedis.publish("news:sports", "Team A won the match!");

四、Spring Data Redis 实现

@Configuration
public class RedisConfig {
    
    @Bean
    RedisMessageListenerContainer container(
            RedisConnectionFactory factory,
            MessageListenerAdapter listenerAdapter) {
        RedisMessageListenerContainer container = 
            new RedisMessageListenerContainer();
        container.setConnectionFactory(factory);
        container.addMessageListener(listenerAdapter, 
            new PatternTopic("chat.*"));
        return container;
    }
    
    @Bean
    MessageListenerAdapter listenerAdapter(RedisReceiver receiver) {
        return new MessageListenerAdapter(receiver, "receive");
    }
}

@Component
public class RedisReceiver {
    public void receive(String message) {
        System.out.println("收到: " + message);
    }
}

@Service
public class MessagePublisher {
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    public void publish(String channel, String message) {
        redisTemplate.convertAndSend(channel, message);
    }
}

五、Pub/Sub 特点

graph LR
    P1[发布者] -->|消息| C[频道]
    P2[发布者] -->|消息| C
    C -->|广播| S1[订阅者1]
    C -->|广播| S2[订阅者2]
    C -->|广播| S3[订阅者3]
  • 实时性：消息即时推送
  • 无持久化：订阅者离线期间消息丢失
  • 无确认：发布者不知道消息是否被接收
  • 广播：所有订阅者都收到相同消息

六、应用场景

1. 实时消息推送

// 聊天室
jedis.publish("chat:room:1001", 
    "{\"user\":\"Alice\",\"msg\":\"Hello!\"}");

2. 配置刷新通知

// 配置中心修改后通知应用刷新
jedis.publish("config:refresh", "app.properties");

3. 日志广播

jedis.publish("log:warn", "Disk space low on server-01");

特性Pub/SubListStream
持久化否是是
离线消息不支持支持支持
消费确认无无ACK 确认
消费者组无无支持
适用场景实时广播简单队列可靠消息队列

八、注意事项

  • 订阅连接是阻塞的，不能执行其他命令
  • 频道名区分大小写
  • 大数据量时考虑使用 Stream 替代
  • 客户端断线需自动重连
