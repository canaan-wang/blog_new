Redis Sentinel 高可用

一、Sentinel 是什么

Redis Sentinel 是监控、通知、自动故障转移的系统，为 Redis 主从架构提供高可用方案。

二、架构组成

graph TD
    subgraph "Sentinel 集群"
        S1[Sentinel 1]
        S2[Sentinel 2]
        S3[Sentinel 3]
    end
    
    M[Master]
    R1[Replica 1]
    R2[Replica 2]
    
    S1 -.->|监控| M

发现主节点| S1

三、Sentinel 功能

功能说明
监控持续检查主从节点状态
通知节点故障时发送通知
自动故障转移主节点故障时自动切换
配置提供向客户端提供当前主节点地址

四、配置部署

# sentinel.conf
port 26379
sentinel monitor mymaster 127.0.0.1 6379 2
sentinel down-after-milliseconds mymaster 5000
sentinel parallel-syncs mymaster 1
sentinel failover-timeout mymaster 10000

# 启动
redis-sentinel sentinel.conf
# 或
redis-server sentinel.conf --sentinel

五、关键配置说明

配置项说明
monitor监控的主节点，quorum=2 表示至少2个 Sentinel 同意才判定故障
down-after-milliseconds多少毫秒无响应判定为主观下线
parallel-syncs故障转移时同时同步的从节点数
failover-timeout故障转移超时时间

六、故障转移流程

graph TD
    A[Sentinel 检测到 Master 无响应] --> B[主观下线 SDOWN]
    B --> C{其他 Sentinel 确认?}
    C -->|是| D[客观下线 ODOWN]
    D --> E[选举 Leader Sentinel]
    E --> F[选择最优 Slave]
    F --> G[提升为 Master]
    G --> H[通知其他 Slave 切换]
    H --> I[通知客户端]

七、Java 客户端连接

// Jedis Sentinel
Set sentinels = new HashSet<>();
sentinels.add("127.0.0.1:26379");
sentinels.add("127.0.0.1:26380");
sentinels.add("127.0.0.1:26381");

JedisSentinelPool pool = new JedisSentinelPool(
    "mymaster",      // Sentinel 监控的名称
    sentinels,
    poolConfig,
    5000             // 连接超时
);

try (Jedis jedis = pool.getResource()) {
    jedis.set("key", "value");
}

八、Spring Data Redis 配置

@Bean
public RedisConnectionFactory redisConnectionFactory() {
    RedisSentinelConfiguration sentinelConfig = 
        new RedisSentinelConfiguration()
            .master("mymaster")
            .sentinel("127.0.0.1", 26379)
            .sentinel("127.0.0.1", 26380)
            .sentinel("127.0.0.1", 26381);
    
    return new JedisConnectionFactory(sentinelConfig);
}

九、Sentinel 命令

# 查看监控的主节点
redis-cli -p 26379 SENTINEL masters

# 查看从节点
redis-cli -p 26379 SENTINEL slaves mymaster

# 查看 Sentinel 信息
redis-cli -p 26379 SENTINEL sentinels mymaster

# 手动故障转移
redis-cli -p 26379 SENTINEL failover mymaster

# 检查状态
redis-cli -p 26379 SENTINEL ckquorum mymaster

十、注意事项

  • 至少部署 3 个 Sentinel 节点
  • Sentinel 节点应分布在不同物理机
  • 网络分区时需 quorum 判断避免脑裂
  • 故障转移期间会有短暂不可用
  • 客户端需支持 Sentinel 协议
