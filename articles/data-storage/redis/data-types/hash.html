Redis Hash

一、Hash 类型简介

Hash 是键值对集合，适合存储对象。每个 Hash 可以存储 2^32 - 1 个键值对。

graph LR
    A[user:1] --> B[name: Tom]
    A --> C[age: 25]
    A --> D[email: tom@example.com]

二、基础命令

设置与获取

HSET user:1 name "Tom" age 25 email "tom@example.com"
HGET user:1 name                 # 获取单个字段
HMGET user:1 name age            # 获取多个字段
HGETALL user:1                   # 获取所有字段
HKEYS user:1                     # 获取所有字段名
HVALS user:1                     # 获取所有字段值
HLEN user:1                      # 获取字段数量

增量操作

HINCRBY user:1 age 1             # 字段值增加
HINCRBYFLOAT product:1 price 1.5 # 浮点数增加

其他操作

HDEL user:1 age                  # 删除字段
HEXISTS user:1 name              # 判断字段是否存在
HSETNX user:1 name "Jerry"       # 字段不存在才设置

三、应用场景

1. 存储对象

// 存储用户信息
Map user = new HashMap<>();
user.put("name", "Tom");
user.put("age", "25");
user.put("email", "tom@example.com");
jedis.hset("user:1", user);

// 获取用户
Map userInfo = jedis.hgetAll("user:1");

2. 购物车

// 添加商品到购物车
jedis.hset("cart:user:1", "product:1001", "2");  // 商品ID: 数量
jedis.hset("cart:user:1", "product:1002", "1");

// 获取购物车所有商品
Map cart = jedis.hgetAll("cart:user:1");

// 增加商品数量
jedis.hincrBy("cart:user:1", "product:1001", 1);

// 删除商品
jedis.hdel("cart:user:1", "product:1001");

// 获取商品数量
String count = jedis.hget("cart:user:1", "product:1001");

3. 配置存储

HSET app:config timeout 30 maxConnections 100 enableCache true
HGET app:config timeout

四、内部实现

Hash 底层使用两种数据结构：

  • ziplist（压缩列表）：元素少且小时使用，节省内存
  • hashtable（哈希表）：元素多或数据大时使用
# 转换阈值配置（redis.conf）
hash-max-ziplist-entries 512    # 元素数量超过512转hashtable
hash-max-ziplist-value 64       # 单个值超过64字节转hashtable

五、与 String 存储对象对比

对比String（JSON）Hash
存储空间较大较小（压缩列表优化）
字段访问需要整个读取可以单独获取字段
字段更新需要整个重写可以单独更新字段
适用场景简单对象，读多写少复杂对象，频繁更新字段

六、最佳实践

  • 对象字段较多或需要频繁更新单个字段时，优先使用 Hash
  • Hash 的 key 设计：业务:对象类型:id
  • 注意单个 Hash 不要存储过多字段（建议 < 1000）
  • 利用 HINCRBY 实现原子计数
