Redis Stream 流

一、Stream 简介

Redis 5.0 引入的日志型数据结构，支持消息持久化、消费者组和 ACK 确认，适合构建可靠消息队列。

二、基础命令

添加消息

# 自动生成 ID
XADD mystream * sensor-id 1234 temperature 19.8

# 指定 ID（时间戳-序列号）
XADD mystream 1704067200000-0 name "Alice"

# 设置最大长度
XADD mystream MAXLEN ~ 1000 * field value

读取消息

# 范围查询
XRANGE mystream - +              # 读取全部
XRANGE mystream 1704067200000-0 +  COUNT 10  # 从指定 ID 开始，最多10条
XREVRANGE mystream + - COUNT 10  # 倒序读取最新10条

# 阻塞读取（消费者）
XREAD BLOCK 0 STREAMS mystream $  # 阻塞等待新消息
XREAD COUNT 5 STREAMS mystream 0  # 读取 5 条未确认消息

三、消费者组

# 创建消费者组（从开头消费）
XGROUP CREATE mystream mygroup $

# 创建消费者组（消费历史消息）
XGROUP CREATE mystream mygroup 0

# 消费者读取消息
XREADGROUP GROUP mygroup consumer1 COUNT 5 STREAMS mystream >

# 确认消息处理完成
XACK mystream mygroup 1704067200000-0

# 查看待处理消息
XPENDING mystream mygroup

四、Java 实现

// 添加消息
jedis.xadd("mystream", StreamEntryID.NEW_ENTRY, 
    Collections.singletonMap("temperature", "20.5"));

// 消费者组消费
Map streams = new HashMap<>();
streams.put("mystream", StreamEntryID.UNRECEIVED_ENTRY);

List>> result = 
    jedis.xreadGroup("mygroup", "consumer1", 5, 0, false, streams);

// 处理消息并确认
for (StreamEntry entry : result.get(0).getValue()) {
    process(entry.getFields());
    jedis.xack("mystream", "mygroup", entry.getID());
}

五、Spring Data Redis Stream

@Configuration
public class StreamConfig {
    
    @Bean
    public Subscription subscription(RedisConnectionFactory factory) {
        StreamMessageListenerContainer.StreamMessageListenerContainerOptions
            > options = 
            StreamMessageListenerContainerOptions.builder()
                .pollTimeout(Duration.ofSeconds(1))
                .build();
        
        StreamMessageListenerContainer>
            container = StreamMessageListenerContainer.create(factory, options);
        
        Subscription subscription = container.receiveAutoAck(
            Consumer.from("mygroup", "consumer-" + UUID.randomUUID()),
            StreamOffset.create("mystream", ReadOffset.lastConsumed()),
            message -> System.out.println("Received: " + message.getValue())
        );
        
        container.start();
        return subscription;
    }
}

@Service
public class StreamProducer {
    @Autowired
    private StringRedisTemplate template;
    
    public void send(String message) {
        RecordId id = template.opsForStream()
            .add(ObjectRecord.create("mystream", message));
        System.out.println("Sent: " + id);
    }
}

八、消息处理模式

// 1. 至少一次投递 + 幂等处理
public void process(StreamEntry entry) {
    String id = entry.getID().toString();
    if (processedSet.contains(id)) {
        return;  // 已处理，幂等
    }
    // 处理业务
    saveToDB(entry);
    processedSet.add(id);
    jedis.xack("stream", "group", entry.getID());
}

// 2. 死信队列处理
public void checkPending() {
    PendingEntriesResult pending = jedis.xpending(
        "stream", "group", null, null, 100);
    
    for (PendingEntry entry : pending.getPendingEntries()) {
        if (entry.getIdleTime() > 60000) {  // 超过60秒
            // 转移到死信队列
            claimToDLQ(entry);
        }
    }
}

九、注意事项

  • 合理设置 MAXLEN 控制内存使用
  • 及时 ACK 避免消息堆积
  • 处理 Pending 列表中的未确认消息
  • 消费者故障时及时清理空消费者组
  • 监控内存使用，设置合理的过期策略
