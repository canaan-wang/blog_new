Redis 集群模式

一、集群架构

Redis Cluster 采用无中心架构，节点间通过 Gossip 协议通信，数据分片存储。

二、核心特性

特性说明
数据分片16384 个哈希槽分配到各节点
主从复制每个主节点有多个从节点
自动故障转移主节点下线，从节点自动提升
无中心架构节点平等，部分节点失败仍可服务

三、哈希槽计算

// key 的槽位计算
slot = CRC16(key) % 16384

// Hash Tag（强制同槽）
user:{1001}:name  -> slot(CRC16("1001"))
user:{1001}:age   -> slot(CRC16("1001"))  // 同一槽位

四、集群搭建

1. 启动节点

# 配置文件 redis-7000.conf
port 7000
cluster-enabled yes
cluster-config-file nodes-7000.conf
cluster-node-timeout 5000
appendonly yes

# 启动 6 个实例
redis-server redis-7000.conf
redis-server redis-7001.conf
redis-server redis-7002.conf
redis-server redis-7003.conf
redis-server redis-7004.conf
redis-server redis-7005.conf

2. 创建集群

redis-cli --cluster create \
    127.0.0.1:7000 127.0.0.1:7001 127.0.0.1:7002 \
    127.0.0.1:7003 127.0.0.1:7004 127.0.0.1:7005 \
    --cluster-replicas 1

五、集群操作

# 查看集群信息
redis-cli -p 7000 cluster info

# 查看节点信息
redis-cli -p 7000 cluster nodes

# 添加主节点
redis-cli --cluster add-node 127.0.0.1:7006 127.0.0.1:7000

# 添加从节点
redis-cli --cluster add-node 127.0.0.1:7007 127.0.0.1:7000 \
    --cluster-slave --cluster-master-id 

# 重新分片
redis-cli --cluster reshard 127.0.0.1:7000

# 删除节点
redis-cli --cluster del-node 127.0.0.1:7000

六、Java 客户端连接

Jedis Cluster

Set nodes = new HashSet<>();
nodes.add(new HostAndPort("127.0.0.1", 7000));
nodes.add(new HostAndPort("127.0.0.1", 7001));
nodes.add(new HostAndPort("127.0.0.1", 7002));

JedisCluster cluster = new JedisCluster(nodes, 
    5000,  // connectionTimeout
    5000,  // soTimeout
    5,     // maxAttempts
    new GenericObjectPoolConfig<>());

// 使用（自动路由到正确节点）
cluster.set("key", "value");
String value = cluster.get("key");

Spring Data Redis

@Bean
public RedisConnectionFactory redisConnectionFactory() {
    RedisClusterConfiguration config = new RedisClusterConfiguration(
        Arrays.asList("127.0.0.1:7000", "127.0.0.1:7001", "127.0.0.1:7002")
    );
    return new JedisConnectionFactory(config);
}

七、集群限制

  • 多键操作需在同一槽位（使用 Hash Tag）
  • 不支持跨槽事务
  • 批量操作需使用 pipeline 到单个节点
// 使用 Hash Tag 保证同槽
// {user:1001}:name 和 {user:1001}:profile 在同一槽位
jedis.mset("{user:1001}:name", "Alice", 
           "{user:1001}:age", "25");

八、故障转移

    O->>S: 投票
    S->>S: 成为新 Master

九、三种部署模式对比

模式数据量高可用读写性能复杂度
主从复制单节点手动切换读写分离低
Sentinel单节点自动故障转移读写分离中
