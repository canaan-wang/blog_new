InfluxDB 概述

InfluxDB 是目前最流行的时序数据库之一，由 InfluxData 公司开发。本文将介绍 InfluxDB 的基本概念、架构设计和核心特性。

什么是时序数据

时序数据（Time Series Data）是指按照时间顺序记录的数据点集合。每个数据点包含一个时间戳和一个或多个数值。典型的时序数据包括：

  • 服务器 CPU、内存、磁盘使用率
  • 应用程序响应时间和错误率
  • 传感器采集的温度、湿度、压力等
  • 股票价格、交易量等金融数据

InfluxDB 版本演进

InfluxDB 1.x

最初的版本，使用 InfluxQL 查询语言，采用 BoltDB 存储引擎。开源版本支持单节点部署，企业版支持集群。

InfluxDB 2.x

2019 年发布的重大版本更新，主要变化包括：

  • 引入全新的 Flux 查询语言
  • 内置 Chronograf 风格的 UI 界面
  • 统一的数据处理和告警功能
  • 改进的存储引擎（基于 TSM）

InfluxDB 3.x (IOx)

最新的架构重构版本，基于 Apache Arrow 和 Parquet 构建：

  • 使用 Apache Arrow 作为内存格式
  • 使用 Parquet 作为持久化格式
  • 支持 SQL 查询
  • 无限基数（unlimited cardinality）

核心概念

Database（数据库）

与传统数据库类似，用于逻辑隔离不同应用或项目的数据。

Measurement（测量）

相当于关系型数据库中的表，存储同一类时序数据。例如 "cpu_usage"、"temperature"。

Tag（标签）

用于存储元数据的键值对，会被索引以支持快速查询。例如 host=server01、region=us-west。

Field（字段）

存储实际的测量值，不会被索引。例如 value=0.64、temperature=23.5。

Timestamp（时间戳）

数据点的时间标识，默认使用纳秒精度。

Series（序列）

具有相同 measurement、tag set 和 field key 的数据点集合。

数据模型示例


Measurement: cpu_usage
Tags:        host=server01, region=us-west
Fields:      usage_user=0.64, usage_system=0.12
Timestamp:   2024-01-15T08:00:00Z

存储引擎（TSM）

InfluxDB 使用 Time-Structured Merge Tree（TSM）存储引擎：

  • WAL（Write-Ahead Log）：确保数据持久性
  • Cache：内存中的热数据缓存
  • TSM Files：压缩的只读数据文件

生态系统

  • Telegraf：数据收集代理，支持 200+ 种输入插件
  • Chronograf：可视化和仪表板工具（InfluxDB 2.x 已内置）
  • Kapacitor：数据处理引擎，支持实时告警
  • InfluxDB Client Libraries：支持 Go、Python、Java、JavaScript 等语言
