MySQL 分区表

一、分区类型

类型说明适用场景
RANGE范围分区时间范围
LIST列表分区离散值
HASH哈希分区均匀分布
KEY键分区类似 HASH
复合分区子分区大表

二、RANGE 分区

CREATE TABLE logs (
    id BIGINT AUTO_INCREMENT,
    log_time DATETIME,
    message TEXT,
    PRIMARY KEY (id, log_time)
) PARTITION BY RANGE (YEAR(log_time)) (
    PARTITION p2021 VALUES LESS THAN (2022),
    PARTITION p2022 VALUES LESS THAN (2023),
    PARTITION p2023 VALUES LESS THAN (2024),
    PARTITION p_future VALUES LESS THAN MAXVALUE
);

-- 自动分区裁剪
SELECT * FROM logs WHERE log_time >= '2023-01-01';  -- 只扫描 p2023

三、LIST 分区

CREATE TABLE users (
    id INT,
    region VARCHAR(20),
    name VARCHAR(50)
) PARTITION BY LIST COLUMNS(region) (
    PARTITION p_north VALUES IN ('Beijing', 'Tianjin'),
    PARTITION p_east VALUES IN ('Shanghai', 'Jiangsu'),
    PARTITION p_south VALUES IN ('Guangdong', 'Fujian'),
    PARTITION p_other VALUES IN (DEFAULT)
);

四、HASH 分区

CREATE TABLE orders (
    order_id BIGINT,
    user_id INT,
    amount DECIMAL(10,2)
) PARTITION BY HASH(user_id) PARTITIONS 4;

-- 数据均匀分布到 4 个分区

五、分区管理

-- 添加分区
ALTER TABLE logs ADD PARTITION (
    PARTITION p2024 VALUES LESS THAN (2025)
);

-- 删除分区（连同数据）
ALTER TABLE logs DROP PARTITION p2021;

-- 拆分分区
ALTER TABLE logs REORGANIZE PARTITION p_future INTO (
    PARTITION p2024 VALUES LESS THAN (2025),
    PARTITION p_future VALUES LESS THAN MAXVALUE
);

-- 合并分区
ALTER TABLE logs REORGANIZE PARTITION p2021, p2022 INTO (
    PARTITION p_old VALUES LESS THAN (2023)
);

-- 查看分区信息
SELECT PARTITION_NAME, TABLE_ROWS 
FROM INFORMATION_SCHEMA.PARTITIONS 
WHERE TABLE_NAME = 'logs';

六、分区优势

  • 查询优化：分区裁剪，只扫描相关分区
  • 快速删除：DROP PARTITION 比 DELETE 快得多
  • 并行处理：分区级并行查询
  • 独立管理：可单独备份、恢复分区

七、注意事项

  • 分区键必须是主键或唯一键的一部分
  • 分区数不宜过多（建议 < 1024）
  • 定期维护（删除旧分区，添加新分区）
  • 分区键选择影响查询性能
