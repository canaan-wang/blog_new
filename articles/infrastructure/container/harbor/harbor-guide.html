Harbor 企业级镜像仓库完全指南

Harbor 是由 VMware 开源的企业级容器镜像仓库，提供安全、可信的镜像存储与管理能力。作为 CNCF 毕业项目，Harbor 在镜像安全扫描、访问控制、复制分发等方面提供了生产级的解决方案，是企业私有容器平台的理想选择。

一、Harbor 简介

Harbor 于 2014 年由 VMware 中国团队发起，2018 年捐赠给 CNCF，2020 年成为 CNCF 毕业项目。它解决了 Docker Hub 等公有仓库在企业环境中的诸多限制：

  • 安全性：支持镜像漏洞扫描、签名验证、内容信任
  • 可控性：私有化部署，数据完全自主掌控
  • 合规性：RBAC 权限管理、操作审计日志
  • 可靠性：多副本复制、高可用架构

5. 自签名证书配置（测试环境）

# 生成 CA 私钥
openssl genrsa -out ca.key 4096

# 生成 CA 证书
openssl req -x509 -new -nodes -sha512 -days 3650 \
  -subj "/C=CN/ST=Beijing/L=Beijing/O=example/OU=Personal/CN=example.com" \
  -key ca.key \
  -out ca.crt

# 生成服务器私钥
openssl genrsa -out harbor.key 4096

# 生成证书签名请求
openssl req -sha512 -new \
  -subj "/C=CN/ST=Beijing/L=Beijing/O=example/OU=Personal/CN=harbor.example.com" \
  -key harbor.key \
  -out harbor.csr

# 生成 x509 v3 扩展文件
cat > v3.ext <<-EOF
authorityKeyIdentifier=keyid,issuer
basicConstraints=CA:FALSE
keyUsage = digitalSignature, nonRepudiation, keyEncipherment, dataEncipherment
extendedKeyUsage = serverAuth
subjectAltName = @alt_names

[alt_names]
DNS.1=harbor.example.com
DNS.2=harbor
IP.1=192.168.1.100
EOF

# 使用 CA 签发证书
openssl x509 -req -sha512 -days 3650 \
  -extfile v3.ext \
  -CA ca.crt -CAkey ca.key -CAcreateserial \
  -in harbor.csr \
  -out harbor.crt

# 客户端信任 CA 证书
sudo cp ca.crt /etc/docker/certs.d/harbor.example.com/ca.crt
sudo systemctl restart docker

2. 基于角色的访问控制（RBAC）

角色 权限 适用场景
Project Admin 完整权限（管理成员、配置、删除项目） 项目负责人
Maintainer 推送/拉取镜像、管理标签、查看日志 CI/CD 服务账号
Developer 推送/拉取镜像 开发人员
Guest 仅拉取镜像 运维人员、外部团队

3. 镜像漏洞扫描

Harbor 集成 Trivy 实现自动化的镜像安全扫描：

sequenceDiagram
    participant User
    participant Harbor
    participant Trivy
    participant DB

    User->>Harbor: 推送镜像
    Harbor->>Harbor: 触发扫描任务
    Harbor->>Trivy: 调用扫描 API
    Trivy->>DB: 查询漏洞数据库
    Trivy->>Harbor: 返回扫描结果
    Harbor->>Harbor: 存储报告
    User->>Harbor: 查看扫描报告

扫描策略配置：

  • 自动扫描：镜像推送时自动触发扫描
  • 定时扫描：按设定时间周期重新扫描所有镜像
  • 阻断策略：根据漏洞等级阻止镜像拉取

漏洞等级：

  • 🔴 Critical（严重）
  • 🟠 High（高危）
  • 🟡 Medium（中危）
  • 🟢 Low（低危）

六、CI/CD 集成

1. Jenkins 集成

pipeline {
    agent any
    environment {
        HARBOR_URL = 'harbor.example.com'
        HARBOR_PROJECT = 'production'
        IMAGE_NAME = 'myapp'
    }
    stages {
        stage('Build') {
            steps {
                script {
                    docker.build("${HARBOR_URL}/${HARBOR_PROJECT}/${IMAGE_NAME}:${BUILD_NUMBER}")
                }
            }
        }
        stage('Scan') {
            steps {
                script {
                    // 等待 Harbor 自动扫描完成
                    sleep(time: 30, unit: 'SECONDS')

                    // 检查扫描结果
                    def scanResult = sh(
                        script: """
                        curl -s -u admin:Harbor12345 \
                        "https://${HARBOR_URL}/api/v2.0/projects/${HARBOR_PROJECT}/repositories/${IMAGE_NAME}/artifacts/${BUILD_NUMBER}/additions/vulnerabilities"
                        """,
                        returnStdout: true
                    )

                    // 解析漏洞数量
                    def critical = sh(
                        script: "echo '${scanResult}' | grep -o 'Critical' | wc -l",
                        returnStdout: true
                    ).trim()

                    if (critical.toInteger() > 0) {
                        error "发现严重漏洞，阻断构建"
                    }
                }
            }
        }
        stage('Push') {
            steps {
                script {
                    docker.withRegistry("https://${HARBOR_URL}", 'harbor-credentials') {
                        docker.image("${HARBOR_URL}/${HARBOR_PROJECT}/${IMAGE_NAME}:${BUILD_NUMBER}").push()
                        docker.image("${HARBOR_URL}/${HARBOR_PROJECT}/${IMAGE_NAME}:${BUILD_NUMBER}").push('latest')
                    }
                }
            }
        }
    }
}

#!/bin/bash
# backup.sh

BACKUP_DIR="/backup/harbor/$(date +%Y%m%d)"
mkdir -p $BACKUP_DIR

# 备份 PostgreSQL
docker exec harbor-db pg_dump -U postgres registry > $BACKUP_DIR/registry.sql
docker exec harbor-db pg_dump -U postgres notary_server > $BACKUP_DIR/notary_server.sql
docker exec harbor-db pg_dump -U postgres notary_signer > $BACKUP_DIR/notary_signer.sql

# 保留最近 30 天备份
find /backup/harbor -type d -mtime +30 -exec rm -rf {} \;

2. 灾难恢复

#!/bin/bash
# restore.sh

BACKUP_DIR="/backup/harbor/20240101"

# 停止 Harbor
cd /opt/harbor && docker compose down

# 恢复数据库
docker run --rm -v $BACKUP_DIR:/backup \
  -e PGUSER=postgres \
  -e PGPASSWORD=root123 \
  --network harbor_harbor \
  goharbor/harbor-db:v2.11.0 \
  sh -c 'psql < /backup/registry.sql'

# 恢复镜像数据
tar xzf $BACKUP_DIR/registry.tar.gz -C /

# 重启 Harbor
docker compose up -d

十、最佳实践

1. 安全实践

  • 启用 HTTPS：生产环境必须配置 TLS 证书
  • 强密码策略：管理员密码至少 12 位，包含大小写字母、数字和特殊字符
  • 机器人账户：CI/CD 使用机器人账户而非管理员账户
  • 定期扫描：启用自动扫描，设置漏洞阻断策略
  • 镜像签名：关键镜像启用 Notary 签名验证
  • 定期轮换：定期更换密码和机器人账户 Token

2. 存储管理

  • 配额限制：为每个项目设置合理的存储配额
  • 保留策略：配置镜像保留规则，自动清理旧版本
  • 垃圾回收：定期执行垃圾回收释放空间
  • 存储选型：大规模部署使用 S3/MinIO 等对象存储

3. 高可用设计

  • 数据库高可用：使用 PostgreSQL 主从 + 自动故障转移
  • Redis 高可用：配置 Redis Sentinel 或 Cluster
  • 负载均衡：多 Harbor 实例前置负载均衡器
  • 跨机房复制：配置镜像复制实现异地容灾

4. 运维监控

  • 健康检查：配置健康检查端点监控
  • 资源监控：监控 CPU、内存、磁盘、网络
  • 业务监控：监控镜像推送/拉取成功率
  • 日志收集：集中收集和分析 Harbor 日志
  • 告警配置：设置磁盘空间、扫描失败等告警

十一、常见问题

1. 登录失败 502 Bad Gateway

可能原因及解决方案：

  • Harbor 服务未完全启动：docker compose ps 检查状态
  • 数据库连接失败：检查 PostgreSQL 状态和配置
  • 磁盘空间不足：清理磁盘或扩展存储

2. 推送镜像 413 Request Entity Too Large

调整 Nginx 上传限制：

# common/config/nginx/nginx.conf
client_max_body_size 0;  # 取消限制

3. 镜像扫描失败

检查 Trivy 数据库更新：

# 手动更新漏洞数据库
docker exec -it harbor-trivy trivy image --download-db-only

# 检查 Trivy 日志
docker logs harbor-trivy

4. 忘记管理员密码

# 重置 admin 密码
cd /opt/harbor
docker compose down
