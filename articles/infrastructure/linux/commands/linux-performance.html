Linux 性能分析

一、CPU 分析

查看 CPU 使用率

# 整体情况
top
htop

# 详细统计
mpstat -P ALL 1

# 进程 CPU 占用
pidstat 1
ps -eo pid,comm,%cpu --sort=-%cpu | head

# 负载
uptime
cat /proc/loadavg

CPU 上下文切换

vmstat 1
# cs 列：上下文切换次数

pidstat -w 1
# cswch/s：自愿切换
# nvcswch/s：非自愿切换

二、内存分析

# 内存使用情况
free -h
cat /proc/meminfo

# 进程内存
ps aux --sort=-%mem | head
pidstat -r 1

# 详细内存映射
pmap -x PID

# 内存分配跟踪
valgrind --tool=memcheck ./program

Buffer/Cache

# 查看缓存
free -h
# buff/cache：文件系统缓存

# 清理缓存（谨慎使用）
sync && echo 3 > /proc/sys/vm/drop_caches

三、磁盘 IO 分析

# IO 统计
iostat -x 1
# await：平均 IO 等待时间（ms）
# %util：磁盘利用率

# 进程 IO
pidstat -d 1
iotop

# 详细的 IO 跟踪
strace -e trace=open,close,read,write -p PID

# 文件系统使用率
df -h
du -sh /path

查找大文件

# 查找大于 100M 的文件
find / -type f -size +100M -exec ls -lh {} \;

# 目录大小排序
du -h --max-depth=1 /var | sort -hr | head

四、网络分析

# 连接状态统计
netstat -ant | awk '{print $6}' | sort | uniq -c | sort -rn
ss -s

# 实时流量
iftop
nload

# 抓包分析
tcpdump -i eth0 -w capture.pcap
wireshark capture.pcap

# 端口监听
netstat -tlnp
ss -tlnp

# 连接数统计
netstat -ant | grep ESTABLISHED | wc -l

五、综合分析工具

perf（性能剖析）

# 记录性能数据
perf record -g ./program

# 查看报告
perf report

# 实时统计
perf top

# 查看调用链
perf record -e cpu-clock --call-graph dwarf ./program

火焰图生成

# 1. 使用 perf 采集
git clone https://github.com/brendangregg/FlameGraph
cd FlameGraph

# 采集 30 秒
perf record -F 99 -a -g -- sleep 30
perf script | ./stackcollapse-perf.pl | ./flamegraph.pl > out.svg

bpftrace / eBPF

# 查看系统调用
bpftrace -e 'tracepoint:raw_syscalls:sys_enter { @[comm] = count(); }'

# 查看文件打开
bpftrace -e 'tracepoint:syscalls:sys_enter_openat { printf("%s %s\n", comm, str(args->filename)); }'

六、JVM 性能分析

# 查看 JVM 进程
jps -lvm

# GC 情况
jstat -gcutil PID 1000

# 堆内存
jmap -heap PID
jmap -histo PID | head -20

# 线程 Dump
jstack PID > thread_dump.txt

# 堆 Dump
jmap -dump:format=b,file=heap.hprof PID

七、常见指标阈值

指标健康值说明
CPU 使用率< 70%持续高负载需优化
Load Average< CPU 核数1分钟/5分钟/15分钟
内存使用率< 80%关注 swap 使用
磁盘 IO 等待< 10%%iowait
IO 延迟(await)< 10msSSD 应更低
上下文切换< 10000/s过高可能进程数过多

八、性能调优思路

graph TD
    A[发现性能问题] --> B[定位瓶颈]
    B --> C{瓶颈类型}
    C -->|CPU| D[优化算法/减少计算]

    C -->|网络| G[减少传输/压缩/批量]
