JVM 类加载机制

一、类加载过程

graph LR
    A[加载 Loading] --> B[验证 Verification]
    B --> C[准备 Preparation]
    C --> D[解析 Resolution]
    D --> E[初始化 Initialization]
    E --> F[使用 Using]
    F --> G[卸载 Unloading]

1. 加载

  • 通过全限定名获取二进制字节流
  • 将静态存储结构转化为方法区运行时数据结构
  • 生成 Class 对象，作为方法区数据的访问入口

2. 验证

  • 文件格式验证：魔数、版本号等
  • 元数据验证：语义分析，如是否有父类
  • 字节码验证：控制流、数据流分析
  • 符号引用验证：能否找到对应类

3. 准备

为类变量分配内存并设置默认初始值：

public static int value = 123;
// 准备阶段：value = 0（默认）
// 初始化阶段：value = 123

4. 解析

将常量池中的符号引用转为直接引用。

5. 初始化

执行类构造器 <clinit> 方法，包括：

  • 静态变量赋值
  • 静态代码块执行

三、双亲委派模型

protected Class loadClass(String name, boolean resolve) {
    // 1. 检查是否已加载
    Class c = findLoadedClass(name);
    if (c == null) {
        // 2. 委托父加载器
        if (parent != null) {
            c = parent.loadClass(name, false);
        } else {
            c = findBootstrapClassOrNull(name);
        }
        // 3. 父加载器无法加载，自己加载
        if (c == null) {
            c = findClass(name);
        }
    }
    return c;
}

优势

  • 避免类的重复加载
  • 保证核心类库安全（防止自定义 java.lang.String）

四、打破双亲委派

场景

  • Tomcat：不同 Web 应用隔离
  • OSGi：模块化热部署
  • JDBC：SPI 机制

方法

  1. 重写 loadClass 方法
  2. 使用线程上下文类加载器 Thread.currentThread().getContextClassLoader()

五、初始化时机

有且仅有以下情况会触发初始化：

  1. new 实例化对象
  2. 访问静态变量（非 final）或静态方法
  3. 反射调用 Class.forName()
  4. 子类初始化触发父类初始化
  5. 执行 main() 方法的类
  6. JDK 8+ 接口定义了 default 方法，实现类初始化

六、被动引用示例

public class SuperClass {
    static {
        System.out.println("SuperClass init");
    }
    public static int value = 123;
}

public class SubClass extends SuperClass {
    static {
        System.out.println("SubClass init");
    }
}

// 被动引用1：通过子类访问父类静态变量
System.out.println(SubClass.value); 
// 输出：SuperClass init，不会触发 SubClass 初始化

// 被动引用2：数组定义
SuperClass[] arr = new SuperClass[10]; // 不会初始化

// 被动引用3：常量
public static final int CONSTANT = 123;
System.out.println(SubClass.CONSTANT); // 编译期常量，不会初始化
