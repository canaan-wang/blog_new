JVM 调优实战

一、调优目标

目标关注点配置方向
高吞吐减少 GC 总时间增大堆、选择 Parallel GC
低延迟减少 GC 停顿G1/ZGC、减小堆
小内存控制内存占用减小各区域大小

二、常用配置模板

Web 应用（4G 内存）

-Xms2g -Xmx2g
-Xmn512m
-XX:MetaspaceSize=128m
-XX:MaxMetaspaceSize=256m
-XX:+UseG1GC
-XX:MaxGCPauseMillis=200
-XX:+HeapDumpOnOutOfMemoryError
-XX:HeapDumpPath=/logs/heapdump.hprof

高并发服务（8G 内存）

-Xms4g -Xmx4g
-Xmn1536m
-XX:SurvivorRatio=8
-XX:+UseConcMarkSweepGC
-XX:CMSInitiatingOccupancyFraction=70
-XX:+UseCMSCompactAtFullCollection

微服务（2G 内存）

-Xms1g -Xmx1g
-XX:+UseParallelGC
-XX:MaxGCPauseMillis=100

三、GC 日志分析

# 开启 GC 日志（JDK 8）
-XX:+PrintGCDetails
-XX:+PrintGCDateStamps
-Xloggc:/var/log/gc.log
-XX:+UseGCLogFileRotation
-XX:NumberOfGCLogFiles=10
-XX:GCLogFileSize=100M

四、常见问题排查

内存泄漏

# 1. 查看老年代增长
jstat -gcutil PID 1000

# 2. 堆转储分析
jmap -dump:live,format=b,file=heap.hprof PID

# 3. 使用 MAT 分析 Dominator Tree

Full GC 频繁

# 原因：
# 1. 老年代空间不足
# 2. 大对象直接进入老年代
# 3. 内存泄漏

# 解决：
# 1. 增大老年代
# 2. 调整晋升阈值 -XX:MaxTenuringThreshold
# 3. 优化代码，减少对象创建

GC 停顿过长

# 解决：
# 1. 切换到 G1/ZGC
# 2. 减小堆大小
# 3. 优化对象生命周期

五、调优步骤

  1. 确定调优目标（吞吐/延迟/内存）
  2. 监控当前状态（GC 频率、停顿时间）
  3. 分析 GC 日志，找出瓶颈
  4. 调整参数，验证效果
  5. 持续监控，迭代优化
