JVM 概述

一、什么是 JVM

JVM（Java Virtual Machine，Java虚拟机）是运行 Java 字节码的虚拟计算机，是 Java 跨平台特性的核心支撑。不同操作系统有各自的 JVM 实现，但执行的是相同的字节码。

    A[Java 源代码 .java] -->|javac 编译| B[字节码 .class]

    C -->|解释执行/JIT| E[Linux 机器码]
    C -->|解释执行/JIT| F[Mac 机器码]

三、类加载机制

类加载过程

graph LR
    A[加载 Loading] --> B[验证 Verification]
    B --> C[准备 Preparation]
    C --> D[解析 Resolution]
    D --> E[初始化 Initialization]
    E --> F[使用 Using]
    F --> G[卸载 Unloading]
  1. 加载：读取 .class 文件，生成 Class 对象
  2. 验证：检查字节码安全性
  3. 准备：为静态变量分配内存，设置默认值
  4. 解析：符号引用转为直接引用
  5. 初始化：执行静态代码块，初始化静态变量

rt.jar]
  • Bootstrap：加载 JAVA_HOME/jre/lib 核心类
  • Extension：加载 ext 目录扩展类
  • Application：加载 classpath 下的类

双亲委派：先委托父加载器加载，父加载器无法加载才自己加载。保证核心类库安全。

四、执行引擎

解释执行与 JIT 编译

  • 解释器：逐行解释字节码，启动快执行慢
  • JIT（Just In Time）：热点代码编译为机器码，执行快
    • 热点检测：方法调用计数器 + 回边计数器
    • C1 编译器：客户端模式，快速编译
    • C2 编译器：服务端模式，深度优化

五、JVM 调优基础

常用参数

# 堆内存设置
-Xms512m          # 初始堆大小
-Xmx2g            # 最大堆大小
-Xmn256m          # 年轻代大小

# 元空间
-XX:MetaspaceSize=128m
-XX:MaxMetaspaceSize=256m

# GC 设置
-XX:+UseG1GC      # 使用 G1 垃圾回收器
-XX:MaxGCPauseMillis=200  # 最大 GC 停顿时间

# 打印 GC 日志
-XX:+PrintGCDetails
-Xloggc:gc.log

七、总结

JVM 是 Java 运行的基础，理解其内存结构、类加载机制和执行原理，对于排查内存溢出、性能调优、理解语言特性都有重要作用。
