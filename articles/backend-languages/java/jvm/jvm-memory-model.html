JVM 内存模型（JMM）

一、什么是 JMM

Java Memory Model 定义了多线程环境下共享变量的访问规则，保证程序在不同平台上有一致的并发语义。

三、内存交互操作

操作作用
lock锁定主内存变量
unlock解锁主内存变量
read从主内存读取到工作内存
load将 read 的值放入工作内存副本
use使用工作内存的值
assign给工作内存变量赋值
store将工作内存值传送到主内存
write将 store 的值写入主内存变量

四、Happens-Before 规则

如果操作 A happens-before B，则 A 的结果对 B 可见。

规则说明
程序次序规则同一个线程中，前面的操作 happens-before 后面的
锁定规则unlock happens-before 后面对同一个锁的 lock
volatile 规则写 volatile happens-before 读 volatile
传递性A happens-before B，B happens-before C，则 A happens-before C
线程启动Thread.start() happens-before 线程的每个动作
线程终止线程的所有操作 happens-before 检测到终止
中断规则interrupt() happens-before 检测到中断
对象终结构造函数执行 happens-before finalize()

五、volatile 语义

public class VolatileExample {
    private volatile boolean flag = false;
    private int value = 0;
    
    public void writer() {
        value = 42;           // 1
        flag = true;          // 2 - volatile 写，之前的操作不会重排序到后面
    }
    
    public void reader() {
        if (flag) {           // 3 - volatile 读，之后的操作不会重排序到前面
            System.out.println(value);  // 4 - 保证看到 42
        }
    }
}

volatile 保证

  • 可见性：一个线程修改，其他线程立即可见
  • 有序性：禁止指令重排序
  • 不保证原子性（复合操作需 synchronized 或 Atomic）

六、重排序

编译器和处理器为提高性能可能重排序指令，但需遵守 as-if-serial 和 happens-before。

// 重排序示例
int a = 1;      // 1
int b = 2;      // 2 可能与 1 重排序
int c = a + b;  // 3 依赖 1、2，不会重排序到前面

七、内存屏障

屏障作用
LoadLoad确保 Load1 在 Load2 之前执行
StoreStore确保 Store1 在 Store2 之前执行
LoadStore确保 Load 在 Store 之前执行
StoreLoad确保 Store 在 Load 之前执行（开销最大）

volatile 读写插入的屏障：

  • 写：StoreStore + StoreLoad
  • 读：LoadLoad + LoadStore

八、long/double 的特殊性

JMM 允许对非 volatile 的 long/double 进行 32 位分次读写（非原子）。但现代 JVM 实现通常是原子的。

九、final 域内存语义

public class FinalExample {
    private final int value;  // final 域
    private int normal;       // 普通域
    
    public FinalExample() {
        value = 42;      // 写 final 域
        normal = 1;      // 写普通域（可能重排序到构造器外）
    }
    
    // 其他线程看到对象时，一定看到 value=42
    // 但可能看不到 normal=1
}

十、实践建议

  • 多线程共享变量用 volatile 或 synchronized
  • 复合操作（i++）用 Atomic 类
  • 避免数据竞争（data race）
  • 使用 java.util.concurrent 工具类
