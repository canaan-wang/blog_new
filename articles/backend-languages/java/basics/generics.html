Java 泛型

一、为什么需要泛型

泛型之前，集合存储的是 Object，取出时需强制类型转换，易出错：

List list = new ArrayList();
list.add("hello");
list.add(123); // 编译不报错
String s = (String) list.get(1); // 运行时 ClassCastException

泛型提供编译期类型检查，避免运行时类型错误：

List list = new ArrayList<>();
list.add("hello");
// list.add(123); // 编译报错！
String s = list.get(0); // 无需强制转换

二、泛型基础

2.1 泛型类

// 定义泛型类
public class Box {
    private T content;
    
    public void set(T content) { this.content = content; }
    public T get() { return content; }
}

// 使用
Box stringBox = new Box<>();
stringBox.set("hello");
String s = stringBox.get();

2.2 泛型方法

public class Util {
    // 泛型方法
    public static  void print(T item) {
        System.out.println(item);
    }
    
    // 泛型返回
    public static  T getFirst(List list) {
        return list.isEmpty() ? null : list.get(0);
    }
}

2.3 泛型接口

public interface Comparable {
    int compareTo(T o);
}

// 实现
public class Student implements Comparable {
    @Override
    public int compareTo(Student other) {
        return this.age - other.age;
    }
}

三、类型通配符

3.1 上界通配符 <? extends T>

表示 T 或 T 的子类，用于读取：

public void printNumbers(List list) {
    for (Number n : list) {
        System.out.println(n);
    }
    // list.add(1); // 编译报错！不能写入
}

List ints = Arrays.asList(1, 2, 3);
printNumbers(ints); // OK

3.2 下界通配符 <? super T>

表示 T 或 T 的父类，用于写入：

public void addNumbers(List list) {
    list.add(1); // OK
    list.add(2); // OK
    // Integer i = list.get(0); // 编译报错！读取只能是 Object
}

List nums = new ArrayList<>();
addNumbers(nums); // OK

3.3 PECS 原则

  • Producer Extends：生产者使用 extends（读取）
  • Consumer Super：消费者使用 super（写入）

四、类型擦除

Java 泛型是编译期特性，运行时类型信息被擦除：

List strList = new ArrayList<>();
List intList = new ArrayList<>();
System.out.println(strList.getClass() == intList.getClass()); // true

// 编译后都变成 List，即类型擦除

五、限制与注意事项

  • 不能创建泛型数组：new T[10] 报错
  • 不能实例化类型参数：new T() 报错
  • 不能用基本类型：List<int> 报错，用 List<Integer>
  • 静态方法不能使用类类型参数
  • 运行时无法获取泛型类型（可通过反射获取签名）

六、最佳实践

  • 优先使用泛型集合，避免原始类型
  • 利用钻石运算符 <> 简化代码
  • 设计 API 时考虑 PECS 原则
  • 泛型命名：T（Type）、E（Element）、K（Key）、V（Value）、N（Number）
