Java 判断与比较

一、== 与 equals 的区别

比较方式比较内容适用类型
==引用地址（基本类型比较值）基本类型、引用类型
equals对象内容引用类型（需正确重写）
// 基本类型
int a = 10, b = 10;
a == b; // true

// 引用类型
String s1 = new String("a");
String s2 = new String("a");
s1 == s2;      // false，不同对象
s1.equals(s2); // true，内容相同

二、equals 与 hashCode 约定

  • equals 相等 → hashCode 必须相等
  • hashCode 相等 → equals 不一定相等（哈希冲突）
// 正确重写示例
public class User {
    private Long id;
    private String name;
    
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        User user = (User) o;
        return Objects.equals(id, user.id);
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(id);
    }
}

三、compareTo 比较

实现 Comparable 接口，定义自然排序：

public class Student implements Comparable {
    private int score;
    
    @Override
    public int compareTo(Student other) {
        return this.score - other.score; // 升序
        // return other.score - this.score; // 降序
    }
}

// 使用
Student s1 = new Student(90);
Student s2 = new Student(80);
s1.compareTo(s2); // 正数，表示 s1 > s2

四、Comparator 定制比较

// Lambda 方式
Comparator byScore = (s1, s2) -> s1.getScore() - s2.getScore();

// 方法引用
Comparator byName = Comparator.comparing(Student::getName);

// 链式比较
Comparator byScoreThenName = Comparator
    .comparing(Student::getScore)
    .thenComparing(Student::getName)
    .reversed(); // 反转

五、Objects 工具类

// 判空比较（避免 NPE）
Objects.equals(a, b);     // 安全 equals
Objects.isNull(obj);      // 是否为 null
Objects.nonNull(obj);     // 是否非 null
Objects.requireNonNull(obj, "不能为空"); // 检查非空

六、switch 比较（Java 14+）

// 传统 switch
switch (day) {
    case MONDAY:
    case FRIDAY:
        System.out.println(" busy");
        break;
    default:
        System.out.println(" free");
}

// Java 14+ switch 表达式
String result = switch (day) {
    case MONDAY, FRIDAY -> "busy";
    case TUESDAY        -> "meeting";
    default             -> "free";
};

// 带 yield 的代码块
int result = switch (score) {
    case 90, 100 -> { yield 5; }
    case 80      -> { yield 4; }
    default      -> { yield 3; }
};

七、最佳实践

  • 引用类型比较内容用 equals，不用 ==
  • 常量放 equals 前面，避免 NPE："expected".equals(variable)
  • 重写 equals 必须重写 hashCode
  • 使用 Objects.equals 进行安全比较
  • Java 8+ 优先使用 Comparator 链式 API
