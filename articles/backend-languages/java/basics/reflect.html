Java 反射

一、什么是反射

反射（Reflection）是 Java 在运行时检查和操作类、方法、字段等成员的能力。通过反射，可以在运行时：

  • 获取类的完整信息（构造器、方法、字段）
  • 创建对象（无需 new）
  • 调用方法（无需直接调用）
  • 访问和修改字段（包括私有成员）

二、获取 Class 对象

// 方式1：类名.class
Class clazz1 = String.class;

// 方式2：对象.getClass()
String s = "hello";
Class clazz2 = s.getClass();

// 方式3：Class.forName()（最常用）
Class clazz3 = Class.forName("java.lang.String");

三、常用反射操作

3.1 创建对象

Class clazz = Class.forName("com.example.User");

// 调用无参构造
Object obj = clazz.newInstance(); // Java 9+ 已废弃

// 调用有参构造
Constructor ctor = clazz.getConstructor(String.class, int.class);
Object obj2 = ctor.newInstance("Alice", 25);

3.2 调用方法

// 获取方法
Method method = clazz.getMethod("sayHello", String.class);

// 调用方法
Object result = method.invoke(obj, "World");

3.3 访问字段

// 获取字段
Field field = clazz.getDeclaredField("name");

// 设置可访问（突破 private）
field.setAccessible(true);

// 获取值
String name = (String) field.get(obj);

// 设置值
field.set(obj, "Bob");

五、反射的优缺点

优点缺点
运行时动态操作性能开销大（绕过 JVM 优化）
解耦，无需硬编码破坏封装性
实现通用框架编译期无法检查类型安全
实现注解处理代码可读性差

七、性能优化

  • 缓存 Class、Method、Field 对象，避免重复反射
  • 使用 setAccessible(true) 跳过访问检查
  • Java 7+ 使用 MethodHandle 替代反射（更快）
  • 关键路径避免反射，编译期生成代码（如 LambdaMetafactory）

八、注意事项

  • 反射可访问私有成员，破坏封装，谨慎使用
  • Java 9+ 模块化可能限制反射访问
  • 反射代码可读性差，需充分注释
