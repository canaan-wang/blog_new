Java synchronized 关键字

一、什么是 synchronized

synchronized 是 Java 内置的同步机制，用于解决多线程环境下的线程安全问题，保证同一时刻只有一个线程执行临界区代码。

二、使用方式

1. 同步实例方法

public class Counter {
    private int count = 0;
    
    // 锁对象：当前实例 this
    public synchronized void increment() {
        count++;
    }
}

2. 同步静态方法

public class Counter {
    private static int count = 0;
    
    // 锁对象：类的 Class 对象
    public static synchronized void increment() {
        count++;
    }
}

3. 同步代码块

public class Counter {
    private int count = 0;
    private final Object lock = new Object();
    
    public void increment() {
        // 可指定锁对象，更灵活
        synchronized (lock) {
            count++;
        }
    }
}

四、底层原理

Monitor 机制

每个 Java 对象都有一个 Monitor（监视器锁），synchronized 通过 Monitor 实现互斥：

graph LR
    A[无锁] -->|线程1获取| B[偏向锁]
    B -->|线程2竞争| C[轻量级锁]
    C -->|竞争激烈| D[重量级锁]
    D -->|线程释放| A

锁升级过程（JDK 1.6+）

  1. 偏向锁：无竞争时，记录线程 ID，同线程进入无需 CAS
  2. 轻量级锁：有竞争时，线程自旋等待，不阻塞
  3. 重量级锁：竞争激烈，线程阻塞，OS 调度

五、可重入性

synchronized 是可重入锁，同一线程可多次获取锁：

public synchronized void methodA() {
    methodB(); // 同一线程，可以再次获取锁
}

public synchronized void methodB() {
    // 执行代码
}

六、内存可见性

synchronized 保证内存可见性：

  • 进入：从主内存刷新变量值
  • 退出：将修改刷新到主内存

八、使用示例

线程安全单例

public class Singleton {
    private static volatile Singleton instance;
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

生产者消费者

public class Buffer {
    private List list = new ArrayList<>();
    private final int capacity = 10;
    
    public synchronized void produce(int value) throws InterruptedException {
        while (list.size() == capacity) {
            wait(); // 满了，等待消费
        }
        list.add(value);
        notifyAll(); // 通知消费者
    }
    
    public synchronized int consume() throws InterruptedException {
        while (list.isEmpty()) {
            wait(); // 空了，等待生产
        }
        int value = list.remove(0);
        notifyAll(); // 通知生产者
        return value;
    }
}

九、注意事项

  • 锁对象要私有化、final，避免被外部修改
  • 锁粒度尽量小，只保护临界区
  • 避免在同步代码块中调用外部方法（可能死锁）
  • 优先使用 synchronized，复杂场景再用 ReentrantLock
  • 注意死锁问题：避免循环等待，按固定顺序获取锁
