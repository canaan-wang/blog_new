Java CyclicBarrier

一、简介

CyclicBarrier 是循环屏障，让一组线程互相等待，到达屏障时被阻塞，直到所有线程都到达后才继续执行。可重复使用（Cyclic）。

二、基本用法

import java.util.concurrent.CyclicBarrier;

public class CyclicBarrierDemo {
    public static void main(String[] args) {
        int parties = 3;
        CyclicBarrier barrier = new CyclicBarrier(parties);
        
        for (int i = 0; i < parties; i++) {
            final int id = i;
            new Thread(() -> {
                try {
                    System.out.println("Thread " + id + " is ready");
                    barrier.await();  // 到达屏障，等待其他线程
                    System.out.println("Thread " + id + " continues");
                } catch (Exception e) {
                    e.printStackTrace();
                }
            }).start();
        }
    }
}

三、核心方法

方法说明
CyclicBarrier(int parties)构造，指定参与线程数
CyclicBarrier(int parties, Runnable barrierAction)构造，带屏障操作
int await()到达屏障并等待
int await(long timeout, TimeUnit unit)超时等待
void reset()重置屏障
boolean isBroken()屏障是否损坏
int getNumberWaiting()当前等待线程数

四、屏障操作（Runnable）

CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("All parties arrived! Barrier action executed.");
});

// 当第3个线程调用 await() 时，先执行 barrierAction，然后所有线程继续

五、应用场景

1. 分阶段计算

public class ParallelMatrixSolver {
    private final CyclicBarrier barrier;
    private final double[][] matrix;
    private final int rows;
    
    public ParallelMatrixSolver(double[][] matrix, int threads) {
        this.matrix = matrix;
        this.rows = matrix.length;
        this.barrier = new CyclicBarrier(threads, () -> {
            System.out.println("Phase completed, validating...");
        });
    }
    
    public void solve() {
        int threads = barrier.getParties();
        for (int i = 0; i < threads; i++) {
            final int threadId = i;
            new Thread(() -> {
                try {
                    for (int phase = 0; phase < 10; phase++) {
                        // 每个线程处理一部分行
                        processRows(threadId, phase);
                        barrier.await();  // 等待其他线程完成当前阶段
                    }
                } catch (Exception e) {
                    e.printStackTrace();
                }
            }).start();
        }
    }
}

2. 多线程分阶段任务

public class MultiPhaseTask {
    public static void main(String[] args) {
        CyclicBarrier barrier = new CyclicBarrier(3);
        
        Runnable task = () -> {
            try {
                // 阶段1：数据加载
                loadData();
                barrier.await();
                
                // 阶段2：数据处理
                processData();
                barrier.await();
                
                // 阶段3：结果保存
                saveResult();
                barrier.await();
                
            } catch (Exception e) {
                e.printStackTrace();
            }
        };
        
        for (int i = 0; i < 3; i++) {
            new Thread(task).start();
        }
    }
}

3. 重复使用的比赛场景

public class RaceGame {
    public static void main(String[] args) {
        CyclicBarrier startLine = new CyclicBarrier(4, () -> 
            System.out.println("Race started!")
        );
        
        for (int round = 1; round <= 3; round++) {
            System.out.println("Round " + round);
            
            for (int i = 0; i < 4; i++) {
                final int player = i;
                new Thread(() -> {
                    try {
                        System.out.println("Player " + player + " ready");
                        startLine.await();
                        run();
                    } catch (Exception e) {
                        e.printStackTrace();
                    }
                }).start();
            }
            
            // 重置用于下一轮
            startLine.reset();
        }
    }
}

六、CountDownLatch vs CyclicBarrier

特性CountDownLatchCyclicBarrier
使用次数一次性可重复使用
等待关系一个/多个等一个/多个互相等待
计数方向减到 0加到指定值
主从关系有主从之分线程对等
异常处理无损坏概念线程中断会损坏屏障

七、注意事项

  • 线程中断会导致屏障损坏（isBroken() = true），需 reset() 恢复
  • await() 返回的是到达索引，0 表示最后一个到达
  • 适合多线程分阶段协作的场景
