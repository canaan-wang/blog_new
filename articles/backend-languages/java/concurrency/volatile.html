Java volatile 关键字

一、volatile 作用

volatile 是轻量级的同步机制，保证变量的：

  1. 可见性：一个线程修改，其他线程立即可见
  2. 禁止指令重排序：防止编译器/CPU 优化改变执行顺序

二、内存可见性问题

多线程环境下，每个线程有本地缓存，变量修改可能不及时同步：

// 问题代码
public class VisibilityProblem {
    private boolean flag = false;
    
    public void writer() {
        flag = true; // 可能只写入线程本地缓存
    }
    
    public void reader() {
        while (!flag) {
            // 可能永远读不到 true
        }
    }
}

三、volatile 保证可见性

// 解决：volatile 强制读写主内存
public class VisibilitySolution {
    private volatile boolean flag = false;
    
    public void writer() {
        flag = true; // 立即刷新到主内存
    }
    
    public void reader() {
        while (!flag) {
            // 每次从主内存读取
        }
    }
}

四、volatile 不保证原子性

// volatile 不保证原子操作
public class VolatileNotAtomic {
    private volatile int count = 0;
    
    // count++ 实际是 3 步操作：
    // 1. 读取 count
    // 2. 加 1
    // 3. 写回 count
    // 多线程可能同时读取相同值，导致丢失更新
    public void increment() {
        count++; // 非线程安全！
    }
}

// 解决：使用 synchronized 或 AtomicInteger
public class AtomicSolution {
    private AtomicInteger count = new AtomicInteger(0);
    
    public void increment() {
        count.incrementAndGet(); // CAS 原子操作
    }
}

五、禁止指令重排序

指令重排序问题

// 单例模式的双重检查锁定
public class Singleton {
    private static Singleton instance;
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton(); // 可能重排序！
                }
            }
        }
        return instance;
    }
}

// new 对象实际分 3 步：
// 1. 分配内存
// 2. 初始化对象
// 3. 引用指向内存
// 可能重排序为 1-3-2，其他线程拿到未初始化的对象

volatile 禁止重排序

public class SafeSingleton {
    private static volatile Singleton instance;
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (SafeSingleton.class) {
                if (instance == null) {
                    instance = new Singleton(); // 禁止重排序
                }
            }
        }
        return instance;
    }
}

六、volatile 内存屏障

volatile 读写插入内存屏障，保证执行顺序：

操作屏障效果
写 volatile前面代码不会重排序到后面
读 volatile后面代码不会重排序到前面

七、适用场景

1. 状态标志位

public class Worker {
    private volatile boolean running = true;
    
    public void shutdown() {
        running = false; // 所有线程可见
    }
    
    public void doWork() {
        while (running) {
            // 工作
        }
    }
}

2. 单例模式（双重检查）

public class Singleton {
    private static volatile Singleton instance;
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

3. 读多写少的计数器

// 只保证可见性，不保证原子性
// 适合一个线程写，多个线程读
public class Counter {
    private volatile int count;
    
    public void set(int value) {
        count = value; // 写操作，单线程写安全
    }
    
    public int get() {
        return count; // 读操作，多线程安全
    }
}

九、总结

  • volatile 只保证可见性和有序性，不保证原子性
  • 适用于：状态标志、单例双重检查、读多写少
  • 不适合：多线程计数、复合操作（i++）
  • 需要原子性时，使用 synchronized 或 Atomic 类
