Java ReentrantLock

一、简介

ReentrantLock 是 java.util.concurrent.locks 包提供的显式锁，相比 synchronized 更灵活：

  • 可中断、可超时、可轮询获取锁
  • 支持公平/非公平模式
  • 支持多条件变量（Condition）

二、基本用法

import java.util.concurrent.locks.ReentrantLock;

public class Counter {
    private final ReentrantLock lock = new ReentrantLock();
    private int count = 0;
    
    public void increment() {
        lock.lock();  // 获取锁
        try {
            count++;
        } finally {
            lock.unlock();  // 必须释放锁
        }
    }
}

三、锁获取方式

可中断获取

public void interruptibleLock() throws InterruptedException {
    lock.lockInterruptibly();  // 可被中断
    try {
        // 业务逻辑
    } finally {
        lock.unlock();
    }
}

超时获取

public boolean tryLockWithTimeout() throws InterruptedException {
    if (lock.tryLock(3, TimeUnit.SECONDS)) {
        try {
            // 获取锁成功
            return true;
        } finally {
            lock.unlock();
        }
    }
    // 超时，获取锁失败
    return false;
}

非阻塞尝试

if (lock.tryLock()) {
    try {
        // 处理业务
    } finally {
        lock.unlock();
    }
} else {
    // 获取失败，做其他处理
}

四、公平锁与非公平锁

ReentrantLock fairLock = new ReentrantLock(true);   // 公平锁
ReentrantLock unfairLock = new ReentrantLock(false); // 非公平锁（默认）
  • 公平锁：按请求顺序获取锁，避免饥饿，吞吐量较低
  • 非公平锁：允许插队，性能更高，默认选择

五、Condition 条件变量

public class BoundedBuffer {
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notFull = lock.newCondition();
    private final Condition notEmpty = lock.newCondition();
    
    private final Object[] items;
    private int putptr, takeptr, count;
    
    public BoundedBuffer(int capacity) {
        items = new Object[capacity];
    }
    
    public void put(T x) throws InterruptedException {
        lock.lock();
        try {
            while (count == items.length) {
                notFull.await();  // 队列满，等待
            }
            items[putptr] = x;
            putptr = (putptr + 1) % items.length;
            count++;
            notEmpty.signal();  // 通知消费者
        } finally {
            lock.unlock();
        }
    }
    
    public T take() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0) {
                notEmpty.await();  // 队列空，等待
            }
            @SuppressWarnings("unchecked")
            T x = (T) items[takeptr];
            takeptr = (takeptr + 1) % items.length;
            count--;
            notFull.signal();  // 通知生产者
            return x;
        } finally {
            lock.unlock();
        }
    }
}

六、synchronized vs ReentrantLock

特性synchronizedReentrantLock
使用方式隐式，自动释放显式，需手动 unlock
可中断否是
超时获取否是
公平锁否可选
条件变量一个多个
性能JDK6+ 优化后接近略优（非公平）

七、最佳实践

  • 优先使用 synchronized，代码更简洁安全
  • 需要高级特性（超时、中断、多条件）时用 ReentrantLock
  • unlock() 必须放在 finally 块
  • 使用 tryLock() 避免死锁
