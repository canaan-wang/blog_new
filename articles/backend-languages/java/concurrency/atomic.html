Java Atomic 原子类

一、简介

Atomic 包提供无锁的原子操作类，基于 CAS（Compare-And-Swap）实现，比 synchronized 性能更高。

二、基本类型原子类

import java.util.concurrent.atomic.AtomicInteger;
import java.util.concurrent.atomic.AtomicLong;
import java.util.concurrent.atomic.AtomicBoolean;

AtomicInteger counter = new AtomicInteger(0);

// 原子操作
int old = counter.getAndIncrement();   // i++
int newVal = counter.incrementAndGet();  // ++i
counter.addAndGet(5);                   // += 5
counter.compareAndSet(0, 1);            // CAS：期望值0，更新值1

三、数组类型原子类

import java.util.concurrent.atomic.AtomicIntegerArray;

AtomicIntegerArray array = new AtomicIntegerArray(10);
array.set(0, 100);
array.incrementAndGet(0);
int val = array.addAndGet(1, 5);

四、引用类型原子类

import java.util.concurrent.atomic.AtomicReference;

AtomicReference userRef = new AtomicReference<>();
userRef.set(new User("Alice"));

// CAS 更新
User old = userRef.get();
User updated = new User("Bob");
boolean success = userRef.compareAndSet(old, updated);

// 带版本号的引用（ABA 问题解决）
import java.util.concurrent.atomic.AtomicStampedReference;
AtomicStampedReference stampedRef = 
    new AtomicStampedReference<>(user, 0);

int[] stampHolder = new int[1];
User current = stampedRef.get(stampHolder);
int currentStamp = stampHolder[0];
stampedRef.compareAndSet(current, newUser, currentStamp, currentStamp + 1);

五、字段更新器

import java.util.concurrent.atomic.AtomicIntegerFieldUpdater;

public class User {
    volatile int age;  // 必须是 volatile
    
    private static final AtomicIntegerFieldUpdater ageUpdater =
        AtomicIntegerFieldUpdater.newUpdater(User.class, "age");
    
    public void incrementAge() {
        ageUpdater.incrementAndGet(this);
    }
}

六、累加器（Java 8+）

import java.util.concurrent.atomic.LongAdder;
import java.util.concurrent.atomic.DoubleAdder;
import java.util.concurrent.atomic.LongAccumulator;

// 高并发计数场景，性能优于 AtomicLong
LongAdder adder = new LongAdder();
adder.increment();
adder.add(10);
long sum = adder.sum();

// 累加器，支持自定义运算
LongAccumulator accumulator = new LongAccumulator(Long::max, 0);
accumulator.accumulate(100);
accumulator.accumulate(50);  // 结果保持 100（max）

七、性能对比

场景推荐方案
低竞争计数AtomicLong
高竞争计数LongAdder
单变量原子操作AtomicInteger/AtomicLong
对象引用更新AtomicReference
避免 ABA 问题AtomicStampedReference

八、CAS 原理与问题

ABA 问题

值从 A→B→A，CAS 认为未变化，但实际已变化。解决方案：

  • AtomicStampedReference：带版本号
  • AtomicMarkableReference：带标记位

自旋开销

CAS 失败会自旋重试，高竞争下消耗 CPU。LongAdder 通过分散热点解决。

九、使用示例

public class AtomicCounter {
    private final LongAdder count = new LongAdder();
    
    public void increment() {
        count.increment();
    }
    
    public long get() {
        return count.sum();
    }
}

// 无锁栈
public class LockFreeStack {
    private final AtomicReference> head = new AtomicReference<>();
    
    public void push(T value) {
        Node newNode = new Node<>(value);
        Node current;
        do {
            current = head.get();
            newNode.next = current;
        } while (!head.compareAndSet(current, newNode));
    }
    
    public T pop() {
        Node current;
        Node next;
        do {
            current = head.get();
            if (current == null) return null;
            next = current.next;
        } while (!head.compareAndSet(current, next));
        return current.value;
    }
    
    private static class Node {
        final T value;
        Node next;
        Node(T value) { this.value = value; }
    }
}
