Java 线程池

一、为什么要用线程池

  • 降低资源消耗：复用线程，减少创建销毁开销
  • 提高响应速度：任务到达立即执行，无需等待创建线程
  • 便于管理：统一分配、调优和监控

二、线程池参数

public ThreadPoolExecutor(
    int corePoolSize,      // 核心线程数
    int maximumPoolSize,   // 最大线程数
    long keepAliveTime,    // 空闲线程存活时间
    TimeUnit unit,         // 时间单位
    BlockingQueue workQueue,  // 任务队列
    ThreadFactory threadFactory,        // 线程工厂
    RejectedExecutionHandler handler    // 拒绝策略
)

参数详解

参数说明
corePoolSize常驻线程数，即使空闲也保留
maximumPoolSize最大线程数，队列满后创建
keepAliveTime非核心线程空闲多久回收
workQueue等待执行的任务队列
threadFactory创建线程的工厂，可设置名称
handler任务被拒绝时的处理策略

三、任务队列

队列类型特点
ArrayBlockingQueue有界队列，防止资源耗尽
LinkedBlockingQueue无界队列，默认 Integer.MAX_VALUE
SynchronousQueue直接提交，不存储任务
PriorityBlockingQueue优先级队列

四、拒绝策略

策略行为
AbortPolicy默认，抛出 RejectedExecutionException
CallerRunsPolicy由调用线程执行任务
DiscardPolicy静默丢弃任务
DiscardOldestPolicy丢弃队列最老的任务，重试提交

六、Executors 工厂方法

// 固定线程数，队列无界
ExecutorService fixed = Executors.newFixedThreadPool(10);

// 单线程，保证任务顺序执行
ExecutorService single = Executors.newSingleThreadExecutor();

// 可缓存，线程数无上限，空闲60秒回收
ExecutorService cached = Executors.newCachedThreadPool();

// 定时任务
ScheduledExecutorService scheduled = Executors.newScheduledThreadPool(10);
scheduled.schedule(() -> {}, 10, TimeUnit.SECONDS);           // 延迟执行
scheduled.scheduleAtFixedRate(() -> {}, 0, 10, TimeUnit.SECONDS); // 固定频率

⚠️ 注意：Executors 创建的线程池有风险，建议使用 ThreadPoolExecutor 手动配置：

  • newFixedThreadPool 和 newSingleThreadExecutor：无界队列，可能 OOM
  • newCachedThreadPool：允许创建无限线程，可能资源耗尽

七、正确使用示例

public class ThreadPoolExample {
    // 推荐手动创建
    private static final ThreadPoolExecutor executor = new ThreadPoolExecutor(
        4,                      // 核心线程数
        8,                      // 最大线程数
        60L, TimeUnit.SECONDS,  // 空闲线程存活时间
        new ArrayBlockingQueue<>(100),  // 有界队列
        new ThreadFactoryBuilder().setNameFormat("pool-%d").build(),
        new ThreadPoolExecutor.CallerRunsPolicy()  // 拒绝策略
    );
    
    public void submitTask(Runnable task) {
        executor.submit(task);
    }
    
    // 优雅关闭
    public void shutdown() {
        executor.shutdown();           // 不再接收新任务
        try {
            if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
                executor.shutdownNow(); // 强制关闭
            }
        } catch (InterruptedException e) {
            executor.shutdownNow();
        }
    }
}

// 自定义监控

    }
    
    @Override

        super.afterExecute(r, t);
        // 任务执行后统计
    }
}

九、最佳实践

  • 使用有界队列防止 OOM
  • 根据任务类型配置线程数：
    • CPU 密集型：核心数 + 1
    • IO 密集型：核心数 × 2 或更多
  • 自定义线程名称便于排查问题
  • 使用 CallerRunsPolicy 作为拒绝策略，保证任务不丢
  • 应用关闭时优雅关闭线程池
  • 使用 CompletableFuture 组合异步任务
