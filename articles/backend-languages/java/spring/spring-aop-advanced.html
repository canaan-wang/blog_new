二、切入点表达式进阶

// execution 语法
execution(修饰符? 返回类型 声明类型? 方法名(参数) 异常?)

// 示例
@Pointcut("execution(public * *(..))")                    // 所有 public 方法
@Pointcut("execution(* set*(..))")                        // 所有 set 方法
@Pointcut("execution(* com.service.AccountService.*(..))") // AccountService 所有方法
@Pointcut("execution(* com.service.*.*(..))")             // service 包所有方法
@Pointcut("execution(* com..*Service.find*(..))")         // 任意子包 Service 的 find 方法
@Pointcut("execution(* *(@com.annotation.Log *))")        // 参数有 @Log 注解的方法

三、组合切入点

@Pointcut("@annotation(com.annotation.Log)")
public void logAnnotation() {}

@Pointcut("execution(* com.service..*.*(..))")
public void serviceLayer() {}

// && 交集
@Pointcut("logAnnotation() && serviceLayer()")
public void loggableService() {}

// || 并集
@Pointcut("execution(* com.controller..*.*(..)) || execution(* com.service..*.*(..))")
public void controllerOrService() {}

// ! 排除
@Pointcut("serviceLayer() && !execution(* com.service.*Service.query*(..))")
public void serviceExceptQuery() {}

四、通知参数绑定

@Aspect
@Component
public class LogAspect {
    
    // 绑定 JoinPoint
    @Before("serviceLayer()")
    public void before(JoinPoint jp) {
        String methodName = jp.getSignature().getName();
        Object[] args = jp.getArgs();
        Object target = jp.getTarget();
    }
    
    // 绑定注解
    @Before("@annotation(log)")
    public void beforeWithAnnotation(JoinPoint jp, Log log) {
        System.out.println(log.value());  // 获取注解属性
    }
    
    // 绑定返回值
    @AfterReturning(pointcut = "serviceLayer()", returning = "result")
    public void afterReturn(JoinPoint jp, Object result) {
        System.out.println("Return: " + result);
    }
    
    // 绑定异常
    @AfterThrowing(pointcut = "serviceLayer()", throwing = "ex")
    public void afterThrow(JoinPoint jp, Exception ex) {
        System.out.println("Exception: " + ex.getMessage());
    }
}

五、环绕通知

@Around("serviceLayer()")
public Object around(ProceedingJoinPoint pjp) throws Throwable {
    long start = System.currentTimeMillis();
    
    try {
        // 前置处理
        Object[] args = pjp.getArgs();
        // 可修改参数
        
        // 执行目标方法
        Object result = pjp.proceed(args);
        
        // 后置处理
        // 可修改返回值
        return result;
        
    } catch (Exception e) {
        // 异常处理
        throw e;
    } finally {
        long elapsed = System.currentTimeMillis() - start;
        System.out.println(pjp.getSignature() + " cost: " + elapsed + "ms");
    }
}

为目标类添加新接口实现：

public interface Auditable {
    void setCreatedBy(String user);
    String getCreatedBy();
}

@Aspect
@Component
public class AuditableAspect {
    
    @DeclareParents(value = "com.service.*Service+", 
                    defaultImpl = AuditableImpl.class)

}

八、AOP 执行顺序

graph TD
    A[调用] --> B[@Around before]
    B --> C[@Before]
    C --> D[目标方法]
    D --> E[@AfterReturning
或
@AfterThrowing]
    E --> F[@After]
    F --> G[@Around after]

多个切面的顺序：

@Order(1)   // 数字越小优先级越高
@Aspect
public class SecurityAspect {}

@Order(2)
@Aspect
public class LogAspect {}

九、AOP 性能优化

  • 切入点表达式尽量精确，减少匹配开销
  • 避免在通知中执行耗时操作
  • 不需要修改返回值时用 @Before/@After
  • 切面对象使用单例
