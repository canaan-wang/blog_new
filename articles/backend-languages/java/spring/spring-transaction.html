Spring 事务管理

一、事务管理方式

声明式事务（推荐）

@Service
public class OrderService {
    
    @Transactional
    public void createOrder(Order order) {
        // 自动开启、提交/回滚事务
        orderDao.save(order);
        inventoryDao.deduct(order.getProductId(), order.getQuantity());
    }
}

编程式事务

@Autowired
private PlatformTransactionManager transactionManager;

public void createOrder(Order order) {
    TransactionStatus status = transactionManager.getTransaction(
        new DefaultTransactionDefinition());
    try {
        orderDao.save(order);
        inventoryDao.deduct(order.getProductId(), order.getQuantity());
        transactionManager.commit(status);
    } catch (Exception e) {
        transactionManager.rollback(status);
        throw e;
    }
}

二、@Transactional 参数

参数说明默认值
propagation传播行为REQUIRED
isolation隔离级别DEFAULT
readOnly只读事务false
timeout超时时间（秒）-1
rollbackFor回滚异常RuntimeException
noRollbackFor不回滚异常无

三、事务传播行为

传播行为说明
REQUIRED默认，当前无事务则新建，有则加入
REQUIRES_NEW新建事务，挂起当前事务
NESTED嵌套事务（保存点）
SUPPORTS有事务则加入，无则以非事务执行
NOT_SUPPORTED以非事务执行，挂起当前事务
MANDATORY必须有事务，否则抛异常
NEVER必须无事务，否则抛异常
graph TD
    A[方法A REQUIRED] --> B{已有事务?}
    B -->|是| C[加入事务]
    B -->|否| D[创建新事务]
    
    E[方法B REQUIRES_NEW] --> F[挂起当前事务]
    F --> G[创建独立事务]
    G --> H[恢复挂起事务]

四、事务失效场景

1. 非 public 方法

@Transactional  // 失效！
private void method() {}

2. 同类方法调用

@Service
public class Service {
    public void methodA() {
        methodB();  // 事务失效！绕过了代理
    }
    
    @Transactional
    public void methodB() {}
}

// 解决方案1：注入自身代理
@Autowired
private Service self;

public void methodA() {
    self.methodB();  // 通过代理调用
}

// 解决方案2：拆分到不同类

3. 异常被捕获

@Transactional
public void method() {
    try {
        dao.save();
    } catch (Exception e) {
        // 异常被捕获，事务正常提交！
        log.error(e);
    }
}

4. 异常类型不匹配

// 默认只回滚 RuntimeException
@Transactional  // SQLException 不回滚！
public void method() throws SQLException {
    dao.save();
}

// 正确配置
@Transactional(rollbackFor = Exception.class)

五、事务隔离级别

@Transactional(isolation = Isolation.READ_COMMITTED)
级别脏读不可重复读幻读
READ_UNCOMMITTED可能可能可能
READ_COMMITTED不可能可能可能
REPEATABLE_READ不可能不可能可能
SERIALIZABLE不可能不可能不可能

六、只读事务优化

@Transactional(readOnly = true)
public List getUsers() {
    return userDao.findAll();
}

作用：

  • Spring 不刷新持久化上下文
  • Hibernate 不检测脏数据
  • 某些数据库可优化查询计划

七、分布式事务

// Seata 分布式事务
@GlobalTransactional
public void purchase() {
    orderService.createOrder();      // DB1
    storageService.deduct();         // DB2
    accountService.debit();          // DB3
}

八、最佳实践

  • 事务方法尽量简短，减少锁持有时间
  • 事务内避免 RPC 调用、HTTP 请求
  • 查询操作使用 readOnly = true
  • 明确指定 rollbackFor = Exception.class
  • 避免事务嵌套过深
