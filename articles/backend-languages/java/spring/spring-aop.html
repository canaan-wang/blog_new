Spring AOP 面向切面编程

一、AOP 概念

AOP（Aspect-Oriented Programming）将横切关注点（日志、事务、权限等）从业务逻辑中分离，提高模块化。

二、核心概念

术语说明
Aspect（切面）横切关注点的模块化，包含通知和切点
Join Point（连接点）程序执行点，如方法调用
Pointcut（切点）匹配连接点的表达式
Advice（通知）切点处执行的代码
Target（目标）被代理的对象
Proxy（代理）AOP 框架创建的对象

三、通知类型

注解执行时机
@Before方法执行前
@After方法执行后（无论是否异常）
@AfterReturning方法成功返回后
@AfterThrowing方法抛出异常后
@Around环绕通知，控制方法执行

四、基本用法

@Aspect
@Component
public class LoggingAspect {
    
    // 定义切点
    @Pointcut("execution(* com.service.*.*(..))")
    public void serviceLayer() {}
    
    @Before("serviceLayer()")
    public void logBefore(JoinPoint jp) {
        System.out.println("调用: " + jp.getSignature());
    }
    
    @AfterReturning(pointcut = "serviceLayer()", returning = "result")
    public void logAfter(JoinPoint jp, Object result) {
        System.out.println("返回: " + result);
    }
    
    @Around("serviceLayer()")
    public Object logAround(ProceedingJoinPoint pjp) throws Throwable {
        long start = System.currentTimeMillis();
        Object result = pjp.proceed();
        long elapsed = System.currentTimeMillis() - start;
        System.out.println(pjp.getSignature() + " 耗时: " + elapsed + "ms");
        return result;
    }
}

五、切点表达式

// execution: 匹配方法签名
@Pointcut("execution(* com.service.*.*(..))")        // service 包所有方法
@Pointcut("execution(public * *(..))")               // 所有 public 方法
@Pointcut("execution(* *ById(Long))")                // 以 ById 结尾，参数 Long
@Pointcut("execution(* com..*Service.*(..))")        // com 包及子包的 Service

// within: 匹配类型
@Pointcut("within(com.service.*)")                   // service 包内
@Pointcut("within(@org.springframework.stereotype.Service *)")

// @annotation: 匹配注解
@Pointcut("@annotation(com.annotation.Log)")         // 方法有 @Log 注解
@Pointcut("@within(com.annotation.Log)")             // 类有 @Log 注解

// args: 匹配参数
@Pointcut("args(java.lang.String)")                  // 参数为 String

// bean: 匹配 Bean 名称
@Pointcut("bean(*Service)")                          // 以 Service 结尾

七、事务管理示例

@Configuration
@EnableTransactionManagement
public class TransactionConfig {
    
    @Bean
    public PlatformTransactionManager transactionManager(DataSource ds) {
        return new DataSourceTransactionManager(ds);
    }
}

@Service
public class OrderService {
    
    @Transactional(
        propagation = Propagation.REQUIRED,
        isolation = Isolation.READ_COMMITTED,
        timeout = 30,
        rollbackFor = Exception.class
    )
    public void createOrder(Order order) {
        // 事务自动管理
    }
    
    @Transactional(readOnly = true)
    public List listOrders() {
        // 只读事务，优化性能
    }
}

八、执行顺序

graph TD
    A[调用方法] --> B[@Around before]
    B --> C[@Before]
    C --> D[目标方法]
    D --> E[@AfterReturning
或
@AfterThrowing]
    E --> F[@After]
    F --> G[@Around after]

九、最佳实践

  • 优先使用 Around 通知获取完整控制
  • 切点表达式尽量精确，避免影响过多方法
  • 避免在通知中抛出异常，除非必要
  • 注意自调用问题（this.method() 不会走代理）
  • 复杂的横切关注点才用 AOP，简单场景直接编码
