MyBatis 缓存机制

一、缓存层级

MyBatis 提供两级缓存：

graph TD
    A[查询] --> B{一级缓存
SqlSession}
    B -->|命中| C[返回结果]
    B -->|未命中| D{二级缓存
Namespace}
    D -->|命中| E[返回结果]
    D -->|未命中| F[查询数据库]
    F --> G[存入二级缓存]
    G --> H[存入一级缓存]
    H --> C

二、一级缓存（默认开启）

SqlSession 级别，会话期间有效。

SqlSession session = sqlSessionFactory.openSession();
UserMapper mapper = session.getMapper(UserMapper.class);

// 第一次查询，命中数据库
User user1 = mapper.selectById(1L);

// 同一 SqlSession，命中一级缓存
User user2 = mapper.selectById(1L);
System.out.println(user1 == user2);  // true

session.close();  // 缓存失效

清空时机

  • 执行 commit/rollback
  • 执行 update/insert/delete
  • 调用 clearCache()
  • SqlSession 关闭

3. 注解方式

@CacheNamespace(
    eviction = FifoCache.class,
    flushInterval = 60000,
    size = 512,
    readWrite = false
)
public interface UserMapper {
    @Select("SELECT * FROM user WHERE id = #{id}")
    @Options(useCache = true)
    User selectById(Long id);
}

集成 Redis

public class RedisCache implements Cache {
    private final String id;
    private final JedisPool jedisPool;
    
    public RedisCache(String id) {
        this.id = id;
        this.jedisPool = new JedisPool("localhost");
    }
    
    @Override
    public String getId() { return id; }
    
    @Override
    public void putObject(Object key, Object value) {
        try (Jedis jedis = jedisPool.getResource()) {
            jedis.setex(serialize(key), 3600, serialize(value));
        }
    }
    
    @Override
    public Object getObject(Object key) {
        try (Jedis jedis = jedisPool.getResource()) {
            return deserialize(jedis.get(serialize(key)));
        }
    }
    // ... 其他方法
}

六、缓存失效问题

1. 脏读

二级缓存导致其他 SqlSession 读到旧数据。解决方案：

  • 设置合适的 flushInterval
  • 关联查询使用 共享缓存
  • 关键数据禁用缓存：useCache="false"

2. 关联表更新

七、最佳实践

  • 频繁查询、少修改的数据适合缓存
  • 分布式环境用 Redis 等集中缓存
  • 设置合理的过期时间
  • 大对象不要缓存（内存压力）
  • 关键业务查询强制刷新缓存
