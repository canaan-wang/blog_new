Java Optional

一、Optional 简介

Optional 是 Java 8 引入的容器类，用于表示可能为 null 的值，避免 NullPointerException。

二、创建 Optional

import java.util.Optional;

// 有值
Optional present = Optional.of("value");  // null 会抛 NPE
Optional nullable = Optional.ofNullable(getValue());  // 允许 null

// 空值
Optional empty = Optional.empty();

// 不推荐（违背设计初衷）
Optional bad = Optional.ofNullable(null);

三、常用方法

方法说明空值时行为
isPresent()是否有值返回 false
isEmpty()是否为空（JDK11+）返回 true
get()获取值抛 NoSuchElementException
orElse(T)获取值或默认值返回默认值
orElseGet(Supplier)获取值或延迟提供执行 Supplier
orElseThrow()获取值或抛异常抛异常
or(Supplier)获取值或返回另一 Optional（JDK9+）-
ifPresent(Consumer)有值时消费什么都不做
ifPresentOrElse(Consumer, Runnable)有值消费，无值执行另一操作（JDK9+）-

四、函数式操作

Optional opt = Optional.of("Hello");

// map：转换值
Optional length = opt.map(String::length);  // Optional[5]

// flatMap：避免嵌套 Optional
Optional> nested = opt.map(this::findByName);  // 不好
Optional flat = opt.flatMap(this::findByName);          // 好

// filter：条件过滤
Optional filtered = opt.filter(s -> s.length() > 3);  // Optional[Hello]

// 链式操作
String result = getUserById(id)
    .map(User::getAddress)
    .map(Address::getCity)
    .filter(city -> !city.isEmpty())
    .orElse("Unknown");

五、使用示例

替代 null 返回

// 不推荐
public String findUserName(Long id) {
    User user = userRepo.findById(id);
    return user != null ? user.getName() : null;
}

// 推荐
public Optional findUserName(Long id) {
    return Optional.ofNullable(userRepo.findById(id))
        .map(User::getName);
}

避免嵌套 if

// 不推荐
if (user != null) {
    Address address = user.getAddress();
    if (address != null) {
        String city = address.getCity();
        if (city != null) {
            return city.toUpperCase();
        }
    }
}
return "UNKNOWN";

// 推荐
return Optional.ofNullable(user)
    .map(User::getAddress)
    .map(Address::getCity)
    .map(String::toUpperCase)
    .orElse("UNKNOWN");

六、Optional 最佳实践

推荐避免
作为方法返回值作为字段
链式处理空值isPresent() 后 get()
orElseGet 用于延迟计算orElse 用于固定值
orElseThrow 提供有意义异常直接 get()

七、注意事项

  • Optional 本身也可能为 null（不要这样用）
  • 不要在集合中使用 Optional（用空集合代替）
  • 不要用 == 比较 Optional
  • 序列化：Optional 不支持序列化
