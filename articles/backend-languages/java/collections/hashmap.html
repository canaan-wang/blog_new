Java HashMap

一、HashMap 核心特性

  • 基于哈希表实现，key-value 存储
  • 无序，key 唯一，允许 null key 和 null value
  • 非线程安全（可用 ConcurrentHashMap）
  • 默认初始容量 16，负载因子 0.75

二、底层数据结构

// JDK 1.8：数组 + 链表 + 红黑树
Node[] table; // 哈希桶数组

    final int hash;    // 哈希值
    final K key;       // 键

// 添加/修改

四、哈希算法

// 计算 hash：高 16 位与低 16 位异或，减少冲突
static final int hash(Object key) {
    int h;
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

// 定位桶索引：(n - 1) & hash，相当于 hash % n
int index = (n - 1) & hash;

五、put 流程

  1. 计算 key 的 hash 值
  2. 定位桶索引 (n-1) & hash
  3. 桶为空：直接插入新节点
  4. 桶有值：遍历链表/树
    • key 相同：覆盖 value
    • key 不同：尾插法插入链表
    • 链表长度 ≥ 8：转为红黑树
  5. 检查扩容：size > threshold（容量 × 负载因子）

六、扩容机制

final Node[] resize() {
    Node[] oldTab = table;
    int oldCap = (oldTab == null) ? 0 : oldTab.length;
    int oldThr = threshold;
    int newCap = oldCap << 1; // 容量翻倍
    threshold = newCap * loadFactor;
    
    // 重新哈希，元素分散到新桶
    Node[] newTab = (Node[])new Node[newCap];
    table = newTab;
    // ... 迁移节点
}

扩容特点：

  • 容量翻倍（2 的幂次）
  • 重新计算桶位置：原位置 或 原位置 + 旧容量
  • JDK 1.8 迁移时链表保持顺序，避免死循环

七、链表转红黑树

  • 链表长度 ≥ 8，桶数量 ≥ 64 时转红黑树
  • 红黑树节点数 ≤ 6 时退化为链表
  • 目的：极端哈希冲突时，查询从 O(n) 优化到 O(log n)

八、线程安全方案

// 方式1：Collections 同步包装（全局锁，性能低）
Map syncMap = Collections.synchronizedMap(new HashMap<>());

// 方式2：ConcurrentHashMap（推荐，分段锁/CAS）
Map concurrentMap = new ConcurrentHashMap<>();

// ConcurrentHashMap 优化点：
// - JDK 1.7：分段锁（Segment）
// - JDK 1.8：CAS + synchronized（桶级别锁）

十、最佳实践

  • 预估数据量，指定初始容量 new HashMap<>(容量/0.75f + 1)
  • key 必须正确重写 hashCode 和 equals
  • key 建议使用 String、Integer 等不可变类型
  • 多线程环境使用 ConcurrentHashMap
  • Java 8+ 利用 compute/merge 方法简化操作
