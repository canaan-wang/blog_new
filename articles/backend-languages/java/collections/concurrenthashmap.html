Java ConcurrentHashMap

一、ConcurrentHashMap 简介

线程安全的 HashMap 实现，JDK 1.8 使用 CAS + synchronized，比 Hashtable 和 Collections.synchronizedMap 性能更高。

二、JDK 1.7 vs 1.8

版本实现锁机制
JDK 1.7分段数组（Segment）每个 Segment 独立加锁
JDK 1.8数组 + 链表/红黑树CAS + synchronized（桶级别）

四、核心机制

put 流程

  1. 计算 hash，定位桶索引
  2. 桶为空：CAS 插入新节点
  3. 桶有值：synchronized 锁定桶头节点
    • 链表：遍历插入或覆盖
    • 红黑树：按树规则插入
  4. 检查是否需要扩容或树化

get 流程

  1. 计算 hash，定位桶
  2. 桶头节点就是要找的：直接返回
  3. 遍历链表或红黑树查找

注意：get 不需要加锁，利用 volatile 保证可见性。

五、常用操作

ConcurrentHashMap map = new ConcurrentHashMap<>();

// 添加
map.put("key", 1);
map.putIfAbsent("key", 2);  // key 不存在才添加

// 计算并添加
map.compute("key", (k, v) -> v == null ? 1 : v + 1);
map.computeIfAbsent("key2", k -> 100);
map.computeIfPresent("key", (k, v) -> v + 1);

// 合并
map.merge("key", 1, Integer::sum); // 存在则累加，不存在设为1

// 批量操作
map.forEach(3, (k, v) -> System.out.println(k + "=" + v)); // 并行度3

// 搜索
String result = map.search(3, (k, v) -> v > 10 ? k : null);

// 归约
long sum = map.reduceValuesToLong(3, v -> v, 0, Long::sum);

七、size 计算

JDK 1.8 使用 CounterCell 数组分散计数，减少竞争：

@sun.misc.Contended
static final class CounterCell {
    volatile long value;
    CounterCell(long x) { value = x; }
}

// 最终 size 是所有 CounterCell 值 + baseCount 的和

八、扩容机制

支持多线程协助扩容：

  • 一个线程触发扩容
  • 其他线程检测到正在扩容，帮助迁移数据
  • 使用 sizeCtl 控制并发

九、最佳实践

  • 多线程环境首选 ConcurrentHashMap
  • 单线程环境用 HashMap 性能更好
  • 使用 compute/merge 方法原子更新
  • 批量操作使用 forEach、search、reduce 系列方法
