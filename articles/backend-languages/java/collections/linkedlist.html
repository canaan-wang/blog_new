Java LinkedList

一、LinkedList 核心特性

  • 基于双向链表实现
  • 实现了 List 和 Deque 接口，可作为列表、栈、队列使用
  • 插入/删除快，随机访问慢
  • 非线程安全

二、节点结构

private static class Node {
    E item;           // 数据
    Node next;     // 后继指针
    Node prev;     // 前驱指针
    
    Node(Node prev, E element, Node next) {
        this.item = element;
        this.next = next;
        this.prev = prev;
    }
}

三、常用操作

LinkedList list = new LinkedList<>();

// 添加
list.add("A");           // 尾部添加
list.addFirst("B");      // 头部添加
list.addLast("C");       // 尾部添加
list.push("D");          // 栈操作：头部添加

// 获取
String first = list.getFirst();   // 获取头部
String last = list.getLast();     // 获取尾部
String peek = list.peek();        // 查看头部（不移除）

// 删除
list.remove();         // 删除头部
list.removeFirst();    // 删除头部
list.removeLast();     // 删除尾部
list.pop();            // 栈操作：删除头部
String polled = list.poll();      // 删除并返回头部

// 队列操作
list.offer("E");        // 尾部添加（队列）
String polled2 = list.poll();     // 删除头部（队列）

四、作为栈和队列使用

// 作为栈使用（LIFO）
LinkedList stack = new LinkedList<>();
stack.push("First");
stack.push("Second");
String top = stack.pop(); // "Second"

// 作为队列使用（FIFO）
LinkedList queue = new LinkedList<>();
queue.offer("Alice");
queue.offer("Bob");
String served = queue.poll(); // "Alice"

六、适用场景

  • 频繁在头部/尾部插入删除
  • 需要实现栈或队列
  • 不需要随机访问

七、注意事项

  • 避免使用 get(index) 遍历，使用迭代器
  • 比 ArrayList 占用更多内存（每个节点两个指针）
  • Java 6 以前存在first/last的循环引用问题，已修复
