Java ArrayList

一、ArrayList 核心特性

  • 基于动态数组实现，支持随机访问
  • 有序，可重复，允许 null
  • 非线程安全（可用 Collections.synchronizedList 或 CopyOnWriteArrayList）
  • 默认初始容量 10，扩容时增长 1.5 倍

二、创建方式

// 默认容量 10
List list1 = new ArrayList<>();

// 指定初始容量（大数据量推荐）
List list2 = new ArrayList<>(1000);

// 从其他集合创建
List list3 = new ArrayList<>(Arrays.asList("a", "b"));

// Java 9+ 不可变列表
List list4 = List.of("a", "b", "c"); // 不可修改

三、常用操作

List list = new ArrayList<>();

// 增
list.add("a");           // 末尾添加
list.add(0, "b");        // 指定位置插入
list.addAll(anotherList); // 批量添加

// 删
list.remove("a");        // 删除指定元素
list.remove(0);          // 删除指定索引
list.clear();            // 清空

// 改
list.set(0, "new");      // 修改指定位置

// 查
list.get(0);             // 获取指定索引
list.indexOf("a");       // 查找元素位置
list.contains("a");      // 是否包含
list.size();             // 元素数量
list.isEmpty();          // 是否为空

四、遍历方式

// 方式1：for 循环（最快）
for (int i = 0; i < list.size(); i++) {
    String item = list.get(i);
}

// 方式2：增强 for
for (String item : list) {
    System.out.println(item);
}

// 方式3：Iterator（遍历时删除）
Iterator it = list.iterator();
while (it.hasNext()) {
    String item = it.next();
    if (needRemove(item)) it.remove();
}

// 方式4：Stream API
list.stream()
    .filter(s -> s.startsWith("a"))
    .forEach(System.out::println);

五、扩容机制

// 源码简析
default int DEFAULT_CAPACITY = 10;

grow() {
    int oldCapacity = elementData.length;
    int newCapacity = oldCapacity + (oldCapacity >> 1); // 1.5 倍
    elementData = Arrays.copyOf(elementData, newCapacity);
}

扩容过程：

  1. 创建新数组（原容量的 1.5 倍）
  2. 使用 Arrays.copyOf 复制元素
  3. 丢弃旧数组（GC 回收）

六、性能特点

操作时间复杂度说明
get(index)O(1)随机访问，直接数组索引
addO(1) / O(n)末尾添加 O(1)，扩容时 O(n)
add(index, e)O(n)需要移动元素
removeO(n)需要移动元素
containsO(n)遍历查找

七、与 LinkedList 对比

特性ArrayListLinkedList
底层结构数组双向链表
随机访问O(1) 快O(n) 慢
插入/删除O(n) 慢O(1) 快（已知位置）
内存占用少（只存数据）多（额外指针）
适用场景查询多、随机访问频繁增删头尾

八、线程安全方案

// 方式1：Collections 同步包装（所有方法加锁，性能低）
List syncList = Collections.synchronizedList(new ArrayList<>());

// 方式2：CopyOnWriteArrayList（读多写少，写时复制）
List cowList = new CopyOnWriteArrayList<>();

// 方式3：手动同步
synchronized(list) {
    list.add("item");
}

九、最佳实践

  • 预估数据量，指定初始容量，避免频繁扩容
  • 大量增删考虑 LinkedList 或批量操作
  • 多线程环境使用 CopyOnWriteArrayList 或并发集合
  • 返回集合时返回不可变视图或副本，防止外部修改
  • Java 8+ 优先使用 Stream API 处理集合
