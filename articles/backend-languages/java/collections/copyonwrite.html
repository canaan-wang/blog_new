Java CopyOnWrite 容器

一、简介

CopyOnWrite 容器（CopyOnWriteArrayList、CopyOnWriteArraySet）是读多写少场景下的线程安全集合，读操作无锁。

二、实现原理

graph LR
    A[写入操作] --> B[加锁]
    B --> C[复制新数组]
    C --> D[修改新数组]
    D --> E[替换引用]
    E --> F[释放锁]
    
    G[读取操作] --> H[直接读取
无需加锁]

三、CopyOnWriteArrayList

import java.util.concurrent.CopyOnWriteArrayList;

CopyOnWriteArrayList list = new CopyOnWriteArrayList<>();

// 写操作：复制整个数组
list.add("A");      // 加锁 -> 复制 -> 添加 -> 替换引用
list.remove("B");   // 同上

// 读操作：直接读，无锁
String s = list.get(0);      // 无锁
int size = list.size();      // 无锁
for (String item : list) {   // 迭代无锁，遍历的是快照
    System.out.println(item);
}

四、源码分析

public boolean add(E e) {
    final ReentrantLock lock = this.lock;
    lock.lock();
    try {
        Object[] elements = getArray();
        int len = elements.length;
        // 复制新数组
        Object[] newElements = Arrays.copyOf(elements, len + 1);
        newElements[len] = e;
        // 原子替换
        setArray(newElements);
        return true;
    } finally {
        lock.unlock();
    }
}

public E get(int index) {
    // 读操作无锁
    return get(getArray(), index);
}

五、迭代器特性

CopyOnWriteArrayList list = new CopyOnWriteArrayList<>();
list.add("A");
list.add("B");

for (String s : list) {
    System.out.println(s);
    list.add("C");  // 不会抛 ConcurrentModificationException
}

// 输出 A, B（迭代的是快照，看不到新添加的 C）
System.out.println(list);  // [A, B, C, C]

六、适用场景

场景建议
读多写少（读:写 > 10:1）适合
事件监听器列表适合
配置项列表适合
写多读少不适合，复制开销大
数据量大不适合，复制成本高

七、实际应用

// 事件监听器管理
public class EventManager {
    private final CopyOnWriteArrayList listeners 
        = new CopyOnWriteArrayList<>();
    
    public void addListener(EventListener listener) {
        listeners.add(listener);
    }
    
    public void removeListener(EventListener listener) {
        listeners.remove(listener);
    }
    
    public void fireEvent(Event event) {
        // 读操作无锁，遍历安全
        for (EventListener listener : listeners) {
            listener.onEvent(event);
        }
    }
}

八、与 Collections.synchronizedList 对比

特性CopyOnWriteArrayListSynchronizedList
读锁无有
写操作复制数组加锁
内存占用高（两份数组）正常
数据一致性最终一致强一致
迭代安全性安全（快照）需手动加锁

九、注意事项

  • 写操作会复制整个数组，大数据量时慎用
  • 只能保证最终一致性，实时性要求高的场景不适用
  • 适合存储监听器、配置等相对稳定的数据
  • addIfAbsent 等复合操作非原子
