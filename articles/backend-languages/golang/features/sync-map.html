Go sync.Map 详解

sync.Map 是 Go 1.9 引入的并发安全 map，专为以下场景优化：

  • 读多写少（频繁读取，偶尔写入）
  • 多个 goroutine 并发访问不同 key
  • 需要动态增删 key 的场景

与原生 map + mutex 的对比

特性map + sync.RWMutexsync.Map
适用场景写多读少，key 数量固定读多写少，key 动态变化
API 复杂度需手动加锁解锁内置线程安全，直接调用
读性能高并发读有锁竞争无锁读（read 路径优化）
遍历需加锁提供 Range 方法（安全但无顺序）
len需加锁后计算无内置 Len 方法

核心 API

方法说明示例
Store(key, value)存储键值对（key 存在则覆盖）m.Store("a", 1)
Load(key)获取值，返回值和是否存在标志v, ok := m.Load("a")
LoadOrStore(key, value)获取或存储（不存在才存储）v, loaded := m.LoadOrStore("a", 2)
Delete(key)删除键值对m.Delete("a")
Range(fn)遍历所有键值对（fn 返回 false 停止）m.Range(func(k, v any) bool { ... })

使用示例

基础使用

package main

import (
    "fmt"
    "sync"
)

func main() {
    var m sync.Map

    // 存储
    m.Store("key1", "value1")
    m.Store("key2", 123)

    // 读取
    if v, ok := m.Load("key1"); ok {
        fmt.Println("key1:", v) // key1: value1
    }

    // 读取或存储（不存在才存储）
    v, loaded := m.LoadOrStore("key1", "new_value")
    fmt.Println("value:", v, "loaded:", loaded) // value: value1 loaded: true

    // 删除
    m.Delete("key2")

    // 遍历
    m.Range(func(key, value any) bool {
        fmt.Printf("%s: %v\n", key, value)
        return true // 返回 false 停止遍历
    })
}

并发场景

package main

import (
    "fmt"
    "sync"
    "time"
)

func main() {
    var m sync.Map
    var wg sync.WaitGroup

    // 多个 goroutine 写入
    for i := 0; i < 10; i++ {
        wg.Add(1)
        go func(id int) {
            defer wg.Done()
            m.Store(fmt.Sprintf("key_%d", id), id)
        }(i)
    }

    // 多个 goroutine 读取
    for i := 0; i < 10; i++ {
        wg.Add(1)
        go func(id int) {
            defer wg.Done()
            time.Sleep(10 * time.Millisecond)
            if v, ok := m.Load(fmt.Sprintf("key_%d", id)); ok {
                fmt.Printf("Read: key_%d = %v\n", id, v)
            }
        }(i)
    }

    wg.Wait()
}

底层原理

sync.Map 采用"双 map + 原子操作"设计，优化读多写少场景：

type Map struct {
    mu     sync.Mutex
    read   atomic.Pointer[readOnly] // 只读 map，无锁访问
    dirty  map[any]*entry          // 脏 map，需加锁访问
    misses int                     // 读 miss 计数
}

type readOnly struct {
    m       map[any]*entry
    amended bool // dirty 是否有 read 中没有的数据
}

设计原理

graph TD
    A["读取操作 Load"] --> B{"read map 中有?"}
    B -->|是| C["无锁返回"]
    B -->|否| D["misses++"]
    D --> E{"misses 达到阈值?"}
    E -->|是| F["加锁，将 dirty 提升为 read"]
    E -->|否| G["加锁，从 dirty 读取"]
    
    H["写入操作 Store"] --> I{"read map 中有该 key?"}
    I -->|是| J["CAS 更新 entry"]
    I -->|否| K["加锁，写入 dirty"]
    K --> L["设置 amended=true"]
  • read map：原子指针指向，无锁并发读
  • dirty map：锁保护，存储新写入的数据
  • 提升机制：read miss 达到一定次数后，将 dirty 提升为 read

最佳实践

1. 适用场景选择

// 读多写少，key 动态变化 → 用 sync.Map
var cache sync.Map

// 写多读少，key 固定 → 用 map + RWMutex
type SafeMap struct {
    mu sync.RWMutex
    m  map[string]int
}

2. 类型断言封装

sync.Map 存储的是 any 类型，建议封装类型安全的方法：

type StringMap struct {
    m sync.Map
}

func (sm *StringMap) Store(key string, value string) {
    sm.m.Store(key, value)
}

func (sm *StringMap) Load(key string) (string, bool) {
    v, ok := sm.m.Load(key)
    if !ok {
        return "", false
    }
    return v.(string), true
}

3. 避免频繁 Range

Range 需要遍历整个 map，性能较低，避免在高频路径调用：

// 不推荐：每次请求都遍历
func (sm *StringMap) GetAll() []string {
    var result []string
    sm.m.Range(func(k, v any) bool {
        result = append(result, v.(string))
        return true
    })
    return result
}

总结

维度建议
适用场景读多写少、key 动态变化、并发访问不同 key
不适用场景写多读少、需要 Len()、对 key 有顺序要求
类型安全需自行封装类型断言
性能优化批量操作、避免频繁 Range
