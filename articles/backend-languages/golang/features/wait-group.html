Go sync.WaitGroup 详解

sync.WaitGroup 是 Go 语言中用于等待一组 goroutine 执行完成的同步原语，是并发编程中最常用的同步工具之一。

核心概念

WaitGroup 维护一个计数器，通过以下三个方法控制：

方法作用
Add(delta int)增加计数器值（delta 可为负数）
Done()计数器减 1（等价于 Add(-1)）
Wait()阻塞直到计数器为 0

基本使用

等待多个 goroutine 完成

package main

import (
    "fmt"
    "sync"
    "time"
)

func worker(id int, wg *sync.WaitGroup) {
    defer wg.Done() // 完成时减 1

    fmt.Printf("Worker %d starting\n", id)
    time.Sleep(time.Second)
    fmt.Printf("Worker %d done\n", id)
}

func main() {
    var wg sync.WaitGroup

    for i := 1; i <= 3; i++ {
        wg.Add(1) // 增加计数
        go worker(i, &wg)
    }

    wg.Wait() // 等待所有 goroutine 完成
    fmt.Println("All workers completed")
}

批量任务处理

func processUrls(urls []string) {
    var wg sync.WaitGroup
    results := make(chan string, len(urls))

    for _, url := range urls {
        wg.Add(1)
        go func(u string) {
            defer wg.Done()
            // 处理 URL...
            results <- fmt.Sprintf("Processed: %s", u)
        }(url)
    }

    // 等待所有 goroutine 完成后关闭 channel
    go func() {
        wg.Wait()
        close(results)
    }()

    // 收集结果
    for res := range results {
        fmt.Println(res)
    }
}

使用注意事项

1. Add 必须在 Wait 之前

// 错误：Add 在 goroutine 中，可能 Wait 时还未执行 Add
go func() {
    wg.Add(1) // 错误！
    defer wg.Done()
    // ...
}()
wg.Wait()

// 正确：Add 在启动 goroutine 之前
wg.Add(1)
go func() {
    defer wg.Done()
    // ...
}()
wg.Wait()

2. Done 调用次数不能过多

Done 调用次数超过 Add 的值会导致 panic：

var wg sync.WaitGroup
wg.Add(1)
wg.Done()
wg.Done() // panic: sync: negative WaitGroup counter

3. WaitGroup 不可复制

作为参数传递时必须使用指针：

func worker(wg *sync.WaitGroup) { // 必须用指针
    defer wg.Done()
    // ...
}

4. 与 context 结合使用

func worker(ctx context.Context, id int, wg *sync.WaitGroup) {
    defer wg.Done()
    
    select {
    case <-ctx.Done():
        fmt.Printf("Worker %d cancelled\n", id)
        return
    case <-time.After(time.Second):
        fmt.Printf("Worker %d done\n", id)
    }
}

func main() {
    ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
    defer cancel()

    var wg sync.WaitGroup
    for i := 0; i < 5; i++ {
        wg.Add(1)
        go worker(ctx, i, &wg)
    }
    wg.Wait()
}

底层原理

type WaitGroup struct {
    noCopy noCopy
    state  atomic.Uint64 // 高 32 位：计数器；低 32 位：等待者数量
    sema   uint32        // 信号量
}
  • state：高 32 位存储计数器，低 32 位存储等待 Wait 的 goroutine 数
  • sema：信号量，用于阻塞和唤醒等待的 goroutine

常见模式

限制并发数

func limitedConcurrency(urls []string, maxConcurrent int) {
    var wg sync.WaitGroup
    sem := make(chan struct{}, maxConcurrent) // 信号量

    for _, url := range urls {
        wg.Add(1)
        sem <- struct{}{} // 获取信号量

        go func(u string) {
            defer wg.Done()
            defer func() { <-sem }() // 释放信号量

            // 处理 URL...
            fmt.Println("Processing:", u)
            time.Sleep(time.Second)
        }(url)
    }

    wg.Wait()
}

动态添加任务

func dynamicTasks() {
    var wg sync.WaitGroup
    tasks := make(chan int, 10)

    // 启动固定数量的 worker
    for i := 0; i < 3; i++ {
        wg.Add(1)
        go func(id int) {
            defer wg.Done()
            for task := range tasks {
                fmt.Printf("Worker %d processing task %d\n", id, task)
                time.Sleep(100 * time.Millisecond)
            }
        }(i)
    }

    // 发送任务
    for i := 0; i < 10; i++ {
        tasks <- i
    }
    close(tasks)

    wg.Wait()
}

总结

要点说明
核心作用等待一组 goroutine 完成
三个方法Add、Done、Wait
注意事项Add 在 Wait 前、Done 不过量、不复制、用指针传递
适用场景批量任务、并发控制、任务编排
