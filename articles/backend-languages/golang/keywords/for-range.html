Go for-range 循环详解

for-range 是 Go 语言中用于遍历集合的语法糖，支持数组、切片、字符串、map、channel 等多种类型，是 Go 中最常用的遍历方式。

基本语法

for index, value := range collection {
    // index: 索引/键
    // value: 元素值的副本
}

不同集合的遍历

数组和切片

nums := []int{10, 20, 30}

// 遍历索引和值
for i, v := range nums {
    fmt.Printf("index: %d, value: %d\n", i, v)
}

// 只遍历值（忽略索引）
for _, v := range nums {
    fmt.Println(v)
}

// 只遍历索引
for i := range nums {
    fmt.Println(i)
}

字符串

str := "Hello 世界"

// 按 rune 遍历（支持中文）
for i, r := range str {
    fmt.Printf("index: %d, char: %c\n", i, r)
}
// 输出：
// index: 0, char: H
// index: 1, char: e
// ...
// index: 6, char: 世 (中文占 3 字节，索引跳跃)

Map

m := map[string]int{"a": 1, "b": 2, "c": 3}

// 遍历是无序的
for k, v := range m {
    fmt.Printf("key: %s, value: %d\n", k, v)
}

// 只遍历 key
for k := range m {
    fmt.Println(k)
}

Channel

ch := make(chan int, 3)
ch <- 1
ch <- 2
ch <- 3
close(ch)

// 遍历直到 channel 关闭
for v := range ch {
    fmt.Println(v)
}

重要陷阱

1. value 是副本，不是引用

// 错误：修改 value 不影响原切片
nums := []int{1, 2, 3}
for _, v := range nums {
    v *= 10 // 修改的是副本！
}
fmt.Println(nums) // [1 2 3]，未改变

// 正确：通过索引修改
for i := range nums {
    nums[i] *= 10
}

2. range 的副本问题（遍历期间修改原集合）

nums := []int{1, 2, 3}
for i := 0; i < len(nums); i++ {
    nums = append(nums, nums[i]) // 无限循环！
}

// range 在开始前就确定了遍历次数
for _, v := range nums { // 只遍历最初的长度
    fmt.Println(v)
}

3. 大结构体的性能问题

type BigStruct struct {
    Data [1024]int
}

items := []BigStruct{{}, {}}

// 低效：每次遍历都复制整个结构体
for _, item := range items { // 复制 4KB
    fmt.Println(item.Data[0])
}

// 高效：使用索引或指针
for i := range items {
    fmt.Println(items[i].Data[0])
}

4. Goroutine 闭包陷阱

// 错误：所有 goroutine 共享同一个变量
for i, v := range []int{1, 2, 3} {
    go func() {
        fmt.Println(i, v) // 可能输出相同的值
    }()
}

// 正确：将变量作为参数传递
for i, v := range []int{1, 2, 3} {
    go func(i, v int) {
        fmt.Println(i, v)
    }(i, v)
}

底层原理

for-range 会被编译器转换为传统的 for 循环：

// 原始代码
for i, v := range slice {
    // body
}

// 编译后等价于
for i := 0; i < len(slice); i++ {
    v := slice[i] // 值拷贝
    // body
}

最佳实践

场景建议
只需值for _, v := range
只需索引for i := range
需要修改元素使用索引 for i := range
大结构体使用索引或指针避免拷贝
goroutine 中使用参数传递避免闭包陷阱

总结

for-range 是 Go 中简洁优雅的遍历语法，但要注意：

  • value 是副本，修改不影响原集合
  • 遍历前确定次数，期间添加的元素不会被遍历
  • 大结构体使用索引避免拷贝开销
  • goroutine 中注意闭包变量捕获问题
